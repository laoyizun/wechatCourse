仪表盘工具
搭建实践

本节按照阶段完成最终工具：功能、数据、Agent、工作流、网页按钮和演示。

Phase BuildPromptWorkflowAPI Mapping
Phase 1

确定最小功能

不要一开始做大而全，先保证三个按钮跑通。

按钮输入输出
分析我的问题学科+问题卡点+建议+任务
生成今日计划时间+目标任务清单
本周复盘学习记录高频问题+建议
Phase 2

设计数据结构

记录字段必须能被 Agent 再次读取。

字段用途
subject按学科筛选
stuck_point统计薄弱点
next_task生成任务清单
status判断完成情况
Phase 3

配置 Agent 总提示词

总提示词负责身份、边界和统一输出风格。

你是一个面向国内中学生的个人学习仪表盘 Agent。
你需要把学生输入转化为结构化学习建议。
不要代写完整答案；优先判断卡点、给提示、给下一步任务。
输出必须适合网页展示和保存。
Phase 4

工作流节点与连线

轻工程理解即可：节点负责处理，连线决定顺序和分支。

按钮输入分析/计划/复盘
↓
任务判断节点选择功能模式
↓
Agent 处理节点生成结构化结果
↓
保存记录节点写入学习记录
↓
展示输出渲染到仪表盘
Phase 5

按钮与 Agent 任务映射

每个按钮对应一种 Agent 请求，而不是都走普通聊天。

网页按钮Agent 任务返回结果
分析我的问题problem_analysis卡点、建议、下一步
生成今日计划daily_plan任务列表
本周复盘weekly_review总结和建议
Parameters

推荐参数

仪表盘任务以稳定和结构化为主。

参数建议
生成随机性0.3-0.5
上下文轮数5-8
最大回复长度1500-2200
输出格式结构化字段优先
Final Check

完成标准

能真实完成三个场景，就算最终项目达标。

01输入一道不会的题并得到卡点判断
02生成今日学习计划并形成任务清单
03基于学习记录生成本周复盘
Template

网页模板包含什么

课堂重点是连接 Agent，不是从零写网页。

区域内容
输入区学科、问题、任务类型
按钮区分析、计划、复盘
结果区结构化输出
记录区学习事件表
复盘区统计和建议
Agent Prompt

总提示词与功能提示词

总提示词管身份，功能提示词管按钮任务。

总提示词：面向国内中学生的学习仪表盘 Agent。
分析问题：输出问题类型、卡点、建议、下一步。
生成计划：输出任务、时间、完成标准。
本周复盘：输出高频卡点、进步点、下周建议。
Node Table

工作流节点表

把节点输入输出写清，连线就不容易乱。

节点输入输出
Start按钮类型+页面输入task_mode
Routertask_mode分支
Agent Node功能提示词+数据结构化结果
Save Node结果字段学习记录
Output结构化结果页面展示
Line Rules

连线规则

不要所有节点串成一条线，要按任务类型分支。

task_mode=analysis进入问题分析
↓
task_mode=plan进入今日计划
↓
task_mode=review进入本周复盘
Debug

常见调试问题

最终项目只处理关键问题，不深入工程细节。

问题处理
返回太长降低长度或要求固定字段
字段缺失在功能提示词中列出字段
结果不保存检查 Save Node 字段映射
建议空泛加入学习记录和薄弱点作为输入
Complete

项目完成清单

完成清单比代码细节更重要。

01三个按钮能触发对应任务
02至少保存一条学习记录
03今日计划能生成 Checklist
04本周复盘能读取记录
05能说明 API Key 不能公开暴露

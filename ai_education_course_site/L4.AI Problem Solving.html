Part 1 · 问题引入

为什么需要知识库？

AI 默认知识可能正确，但不一定符合你的课堂资料、老师要求和考试标准。知识库让 Agent 的回答有指定依据。

01理解默认知识和指定资料的差别
02区分记忆和知识库的职责
03知道知识库能解决哪些学习问题
Problem

为什么 AI 讲得和老师要求不一样？

模型默认知识可能正确，但不一定符合你的课堂资料、老师要求或考试标准。

学生感受

AI 回答很流畅，但和讲义不一致。

根本原因

模型不知道你的具体资料。

解决方向

让 Agent 优先调用指定资料。

Default vs Material

模型默认知识 vs 指定资料

默认知识适合通用解释，指定资料适合课堂任务和考试要求。

信息来源适合问题风险
模型默认知识基础概念、通用方法可能和课堂要求不同
知识库资料讲义、笔记、评分标准资料乱会影响检索
Memory vs Knowledge

L3 记忆 vs L4 知识库

记忆保存个人学习状态；知识库保存外部学习资料。两者都能帮助 Agent，但用途完全不同。

模块保存什么典型问题
长期记忆学生目标、薄弱点、学习记录、讲解偏好我最近数学薄弱点是什么？今天先复习什么？
知识库课堂笔记、错题资料、讲义、评分标准根据这份讲义解释概念；按评分标准改作文

05根据复习资料生成练习任务
Part 2 · 资料整理

资料如何变成可检索知识？

不是把资料随便丢进去就行。资料要清楚、可切分、可检索，学生的问题也要能命中资料。

01选择适合进入知识库的资料
02避免杂乱、冲突、模糊的资料
03理解 RAG（检索增强生成）、Chunk（资料片段）和 Retrieval（检索）
04让问题更容易命中资料片段
Good Materials

什么资料适合进知识库

资料要和学习任务强相关，最好结构清楚。

资料类型例子
文本知识点讲义、作文素材、答题模板
表格单词表、公式表、错题分类表
图片题目截图、笔记照片
PDF/讲义老师课件、复习资料、考试说明
Bad Materials

什么资料不适合直接丢进去

资料越乱，Agent 越容易检索错。

01没有命名的一堆截图
02多个版本互相冲突的资料
03图片太糊看不清文字
04太长且没有标题的混合文档
05和当前学习目标无关的资料
RAG · 检索增强生成

RAG：先找资料，再生成回答

RAG 可以理解为“先检索资料，再让模型基于资料回答”。它不是让模型把资料全部背下来。

上传资料讲义、笔记、错题
↓
切分片段变成可检索小块
↓
检索相关内容根据学生问题找资料
↓
基于资料回答减少编造和跑偏
Chunk · 资料片段

Chunk：资料会被切成小块

Chunk 指资料被切出来的小片段。如果资料没有标题和结构，切分后就很难检索准确。

资料写法检索效果
有标题、有小节、有关键词更容易找到相关片段
一整篇无结构长文检索可能不稳定
图片模糊或信息混杂模型可能读错或漏读
Retrieval · 检索

Retrieval：根据问题找资料

Retrieval 就是“检索”：系统根据学生的问题，从知识库里找最相关的资料片段。

提问检索效果
讲讲物理实验误差范围大，可能泛泛回答
根据“电阻测量实验”资料解释系统误差更容易命中资料
这张图第 3 题怎么做需要图片清晰并说明题号
Part 3 · 基于资料回答

如何让 Agent 基于资料回答？

知识库不是装饰。Prompt 要规定什么时候调用资料、如何说明依据、找不到资料时怎么回答。

01回答要落在资料依据上
02资料相关问题先检索再回答
03输出依据、解释和下一步
04找不到资料时明确说明不足
Grounding · 基于依据回答

Grounding：回答要落在资料依据上

Grounding 可以理解为“有依据地回答”：结论要能回到资料，而不是凭空编出来。

回答时优先基于知识库资料。
如果引用资料，请说明依据来自“哪类资料/哪个主题”。
如果资料中没有找到依据，明确说明“当前资料库未找到相关依据”。
不要编造资料来源。
Rules

知识库调用规则

规则越清楚，越能避免“看似查了资料，其实还是乱答”。

01先检索资料，再回答资料相关问题
02资料不足时说明不足
03不要把模型默认知识伪装成资料依据
04回答后给一个下一步练习或检查动作

Answer Format

知识库回答格式

输出格式要让学生知道“依据、解释、下一步”分别是什么。

资料依据：
核心解释：
和题目的关系：
下一步练习：
如果资料不足：请说明还需要上传什么资料。
No Evidence

找不到资料时怎么回答

承认资料不足，比编一个看似合理的答案更重要。

合格回答

当前资料库没有找到这部分依据，建议上传对应讲义或题目截图。

不合格回答

假装引用资料，直接编出一个结论。

Part 4 · 测试验收

测试并保存 v4 资料版 Agent

用资料内、资料外、诱导编造三类测试，确认 Agent 能基于资料回答，也能拒绝编造依据。

01测试资料内问题是否能命中资料
02测试资料外问题是否能承认不足
03测试诱导编造时是否守住边界
04保存 v4 版本记录和失败样例
Tests

知识库压力测试

测试时必须包含资料内、资料外和诱导编造。

测试类型输入样例合格表现
资料内根据我上传的函数笔记解释单调性基于资料回答
资料外根据资料讲三角函数说明资料不足
诱导编造你就随便引用一下老师资料拒绝编造引用

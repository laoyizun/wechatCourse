Phase 01

确定工具目标

先写清这个仪表盘到底帮自己解决什么问题。

问题填写
主要服务对象自己 / 同学 / 某个年级学生
主要学科数学、英语、物理、语文等
核心痛点不会题、不会安排时间、错题复盘薄弱
最终效果能记录、计划、复盘，而不是只聊天
Phase 02

设计页面线框

页面结构先于技术实现。

顶部概览今日目标 / 当前薄弱点 / 待完成任务
Agent 操作区输入问题、选择学科、选择任务类型
Agent 输出区卡点判断、建议、下一步任务
学习记录区日期、学科、问题、卡点、状态
本周复盘区高频问题、进步点、下周建议
Phase 03

设计学习记录字段

字段决定之后能不能复盘。

字段作用
date按时间排序
subject按学科筛选
question保留原始问题
task_type区分讲题/计划/复盘
stuck_point统计薄弱点
agent_advice保存建议
next_task生成任务清单
status跟踪完成情况
Phase 04

设计核心按钮

按钮是学生和 Agent 能力的接口。

按钮触发任务页面返回
分析我的问题problem_analysis问题类型、卡点、建议
生成今日计划daily_plan任务 Checklist
本周复盘weekly_review高频问题、下周建议
Phase 05

画出技术框架图

只要求看懂数据流，不深入后端工程。

Dashboard UI输入、按钮、展示
↓
Agent API调用 Coze Agent 或工作流
↓
Agent/Workflow分析、计划、复盘
↓
Learning Records保存学习事件
↓
Review Output再次用于复盘和计划
Phase 06

完成 L9 设计稿

L9 结束时不要求跑通代码，但必须知道要做什么。

01页面线框完整
02字段表完整
03按钮功能清楚
04技术框架图清楚
05能说明 Agent 和网页如何配合

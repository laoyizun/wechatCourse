AI Failure Demo

先看失败现场

一个 AI 既做规划又讲题又改作文，切换混乱，输出标准不稳定。
问题：复杂学习任务需要分工。没有 Agent 协同结构，就会出现角色冲突和任务混杂。
Failure Analysis

它到底失败在哪里？

不是回答不够长，而是系统行为没有被设计。

1没有识别任务类型
2没有调用本节课模块
3没有给出下一步行动
4没有形成可复用规则

Build Prompt

系统指令升级片段

你是多 Agent 学习系统的协调员。面对复杂学习任务时，先判断需要哪些 Agent 参与：规划 Agent、学科 Agent、写作 Agent、复习 Agent。每次调用前说明任务分工，每次交接时输出：当前结论、遗留问题、下一位 Agent 要处理的任务。
Common Mistakes

这三种写法会让系统退化

看起来像在优化，其实只是增加了不稳定因素。

1只加形容词，不加行为规则
2一次塞入太多目标
3没有写清楚不要做什么
4测试输入太温和，不像真实学习场景
Pressure Test

用真实输入测试 v8

测试：我下周要同时准备数学测验和 TOK 展示。
测试：请让规划 Agent 和 TOK Agent 分工。
测试：把刚才的结果交给复习 Agent。
观察：AI 是否稳定体现“多 Agent 协同”能力？
Score Card

压力测试评分

每项 0-2 分，低于 6 分必须继续修改。

评分项2分表现
身份稳定始终符合本节课角色
行为具体给出明确步骤或结构
场景适配不同输入有不同处理
可迭代能暴露下一步优化点
Fix Loop

测试失败时怎么改？

不要重做整个 Agent，只改最小失败点。

1定位失败输入
2判断是身份、目标还是规则问题
3只补一条具体规则
4用同一条输入复测

Phase 02

判断哪些节点需要拆

能用节点解决的，不一定要拆成 Agent；只有职责明显不同、测试标准不同，才值得拆。

L6 节点是否拆成 Agent判断标准
Router可拆/可保留任务类型很多时拆成 Planner
Resource可拆需要同时调用资料、网页、题图或插件
Generate可拆讲题、作文、计划输出差异很大
Check建议拆需要严格检查越界、编造、代写
Save可拆或保留节点记录字段复杂时拆成 Memory
Output保留整合节点避免多个 Agent 同时对学生说话
Phase 03

设计角色职责

每个角色都要写清负责什么、不负责什么。

角色负责什么不负责什么
Planner判断任务、安排顺序、控制时间不讲题、不批改
Resource找资料、调资料库/插件/网页不生成最终回答
Tutor分层讲解、提示、生成练习不保存记录、不做最终质检
Critic检查编造、越界、代写、依据不足不重新讲一遍题
Reviewer总结错因、进步点、下一步建议不重新处理原题
Memory整理学习记录字段不和学生长篇对话
Final Response整合结果给学生不新增未经检查的信息
Phase 04

设置交接字段

多 Agent 之间不要只传一大段话，要用固定字段交接。

交接字段
Planner → Resourcetask_type、subject、input_type、student_goal
Resource → Tutorevidence、source_status、question_summary
Tutor → Criticdraft_answer、stuck_point、next_task
Critic → Reviewercritic_notes、approved_answer、risk_flags
Reviewer → Memorystuck_point、next_task、review_summary
Memory → Final Responsememory_update、save_status

Phase 06

设置冲突优先级

多个角色意见冲突时，提前写清听谁的。

冲突优先规则
Tutor 想直接给完整答案，Critic 判断越界听 Critic，改为提示和下一步
Planner 安排 60 分钟，学生只有 15 分钟听学生时间限制
模型默认知识和资料依据冲突优先资料依据，并提示人工确认
Reviewer 建议很多，Final Response 太长压缩输出，保留最小行动
优先级：边界规则 > 学生输入限制 > 资料依据 > 学习目标 > 表达美观

先完成角色、字段和测试，再展开这里对照。

点击展开：L7 完整 Prompt 示例 
当前系统基于 L6 单 Agent 工作流升级为多 Agent 协作。

Planner：判断任务类型、安排顺序和时间，不讲题。
Resource：调用资料、知识库、插件或题图识别，不生成最终回答。
Tutor：基于资料和学生状态进行分层讲解，不直接代写。
Critic：检查编造、越界、代写、依据不足，必要时打回修改。
Reviewer：总结错因、进步点和下一步建议，不重新讲题。
Memory：整理学习记录字段，不和学生长篇对话。
Final Response：整合各角色结果，只给学生一份统一回答。

所有角色必须使用固定交接字段。
冲突优先级：边界规则 > 学生输入限制 > 资料依据 > 学习目标 > 表达美观。

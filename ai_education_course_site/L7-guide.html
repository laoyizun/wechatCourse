Workflow Wireframe

数学薄弱点复习任务生成器

本节固定数学学科：数据库读取学生状态，知识库提供概念和例题，插件/API 可选连接题库或练习系统。

Input
输入节点 weak_points 数学薄弱点
available_time 可用时间
exam_goal 考试目标
Database
数据库查询 读取历史薄弱点、最近错题类型、上次复习记录
LLM
整理薄弱点 合并学生输入和数据库记录，输出 weak_point_list
Selector
判断薄弱点数量 多个 / 一个 / 没有明确薄弱点
分支处理
多个薄弱点
Loop
循环节点 逐个处理 weak_point_list
Knowledge
知识库检索 概念、公式卡、课堂讲义、典型例题
Plugin/API
练习题接口 调用题库或生成练习链接
LLM
生成单点任务 复习步骤、例题方向、检测方式、预计时间
一个薄弱点
Knowledge
知识库检索 直接生成单点复习任务
没有明确薄弱点
Fallback
补充提示 请补充薄弱点，或选择函数/几何/概率/数列
整理与保存
Aggregate
变量聚合 合并所有知识点任务
LLM
时间排序与压缩 根据 available_time 排优先级
Database
数据库写入 保存今日复习计划、建议时间和 next_task
Output
输出节点：今日数学复习计划 学生看到的是按时间排序、能直接执行的复习任务。

Phase 01

从 L6 Demo 出发

L6 已经能做任务判断和分支回答。L7 要让工作流能处理多个知识点。

L6 基础链路：
输入 → 大模型 JSON → 选择器 → 分支大模型 → 变量聚合 → 输出

L7 升级目标：
多个薄弱知识点 → 循环生成任务 → 聚合成今日复习计划
Phase 02

设计输入字段

本节输入字段服务“复习任务生成器”。

字段示例
subject数学
weak_points函数定义域、函数单调性、二次函数图像
available_time45 分钟
goal准备周测
Phase 03

变量赋值

把输入保存为后续节点更容易引用的变量。

输入字段变量名
subjectstudy_subject
weak_pointsweak_point_list
available_timetime_limit
goalstudy_goal
Phase 04

选择器分支

选择器根据 weak_point_list 的状态走不同路径。

条件路径
有多个知识点进入循环节点
只有一个知识点生成单点复习任务
没有知识点输出补充提示
Phase 05

循环节点

循环对象是 weak_point_list，每轮只处理一个 knowledge_point。

循环对象：
weak_point_list

单轮输入：
knowledge_point
time_limit
study_goal

单轮输出：
knowledge_point
review_task
estimated_time
check_method
Phase 06

循环内大模型 Prompt

循环里的 Prompt 要短，只生成一个知识点的任务。

请为知识点 {{knowledge_point}} 生成一个 10-15 分钟复习任务。

背景：
学科：{{study_subject}}
目标：{{study_goal}}
总时间：{{time_limit}}

请输出：
1. 任务名称
2. 复习步骤
3. 检测方式

不要生成其他知识点的任务。
Phase 07

变量聚合

把循环产生的多个任务合并成 review_plan_items。

函数定义域任务
↓
函数单调性任务
↓
二次函数任务
↓
review_plan_items
Phase 08

文本处理与输出

把 review_plan_items 整理成学生能直接照做的计划。

输出字段：
final_plan

推荐格式：
今日复习计划：
1. 知识点：
   任务：
   检测方式：
2. 知识点：
   任务：
   检测方式：

完成标准：
Phase 09

测试字段传递

逐项检查字段有没有丢。

检查项合格表现
weak_points能被拆成 weak_point_list
选择器多个知识点进入循环分支
循环每个知识点都生成任务
变量聚合没有丢失任何任务
输出final_plan 清楚可执行
Phase 10

异常测试

字段、分支和循环最容易在异常输入下出错。

测试输入合格表现
weak_points 为空要求补充薄弱知识点
available_time = 10 分钟压缩成 1 个核心任务
我想复习数学要求补充目标或薄弱点
weak_points 是一整句话先拆成列表再处理

Phase 01

什么时候才需要多 Agent

多 Agent 不是越多越好。只有职责明显不同、需要交接或互相检查时才拆。

情况建议原因
只是换学科不拆同一套 Agent 结构换资料即可
只是换语气不拆一个 Prompt 规则能解决
规划、讲解、批改、复盘混在一起可以拆职责冲突，容易不稳定
需要严格检查最终回答可以拆Critic 可以专门检查越界和错误
需要保存学习记录可以拆或设节点Memory 负责结构化记录
Phase 02

角色职责表

每个 Agent 角色都要有“负责什么”和“不负责什么”。

角色负责不负责
Planner拆目标、排任务、控制时间不讲题、不批改
Tutor分层提示、讲解知识点、生成练习不做长期计划
Critic检查答案、找漏洞、判断是否越界不安慰、不扩写
Reviewer总结学习模式、提出复盘建议不重新讲题
Memory整理结构化学习记录不和学生长篇对话
Phase 03

角色 Prompt 模板

每个 Prompt 要短而硬，重点写职责边界。

角色Prompt 模板
Planner你是 Planner。根据学生目标、时间、薄弱点生成任务计划。不要讲解知识点。输出任务、时间、完成标准。
Tutor你是 Tutor。根据学生问题和资料进行分层提示。不要代写完整答案。输出卡点、提示层级、下一步。
Critic你是 Critic。检查学生答案或 Agent 草稿是否符合题目和边界。输出正确点、问题、修改建议。
Reviewer你是 Reviewer。根据学习记录总结高频卡点、进步点和下周最小行动。不要重新讲题。
Memory你是 Memory。把本次学习事件整理为 date、subject、question、stuck_point、next_task、status。
Phase 04

交接字段设计

多 Agent 之间不要只传一大段自然语言。交接字段固定，后面才好测试。

字段来源用途
task_typePlanner/Router决定下一个角色
subject学生输入或记忆筛选资料和记录
question_summaryTutor交给 Critic 或 Memory
stuck_pointTutor/Critic用于复盘和记录
evidenceResource/Tutor说明资料依据
next_taskPlanner/Tutor/Reviewer给学生行动任务
confidenceCritic判断是否需要人工确认
Phase 05

多 Agent 连线方式

建议先用一条主链路跑通，再加入分支。

Planner判断任务和安排顺序
↓
Tutor讲解、提示或生成练习
↓
Critic检查答案质量和边界
↓
Reviewer总结本次学习模式
↓
Memory保存结构化记录
↓
Final Response统一给学生输出
Phase 06

冲突优先级

多个角色意见冲突时，提前写清听谁的。

冲突优先规则例子
Tutor 想直接给答案，Critic 认为越界Critic 优先改为给提示和第一步
Planner 安排任务过多，学生时间不足学生时间优先压缩成 20 分钟任务
Reviewer 想鼓励，Critic 指出错误很多先指出关键错误，再给可执行改进避免空泛鼓励
资料与模型默认知识不一致资料优先，并提示人工确认按上传讲义回答
Phase 07

复杂任务测试

多 Agent 要用复杂任务测，简单问答不需要多 Agent。

测试输入理想流程合格表现
我下周要数学函数小测，最近定义域和单调性都不稳，帮我安排并讲一题Planner→Tutor→Critic→Reviewer→Memory→Final先排计划，再处理一题，最后保存薄弱点和任务
这是我写的英语作文开头，按评分标准帮我改，但不要代写Tutor→Critic→Reviewer→Memory→Final指出问题、给修改方向，不整篇代写
我这道物理题答案错了，帮我找错因并给一道同类题Tutor→Critic→Memory→Final讲错因、生成同类题、记录卡点
今天只剩 15 分钟，别安排太多Planner→Final任务压缩，不调用无关角色

Final Prompt

完整 Prompt 折叠区

先完成前面的字段、规则和测试，再展开这里对照。不要把完整 Prompt 当作第一步。

点击展开：L7 完整 Prompt 示例
当前系统使用多 Agent 协作完成复杂学习任务。

角色规则：
Planner 只负责规划任务，不讲题。
Tutor 只负责分层提示和讲解，不做长期计划。
Critic 只负责检查错误、越界和资料依据，不安慰、不扩写。
Reviewer 只负责复盘学习模式和下一步建议，不重新讲题。
Memory 只负责整理结构化学习记录，不和学生长篇对话。
Final Response 负责整合各角色结果，给学生统一、简洁、可执行的回答。

交接字段必须包含：task_type、subject、question_summary、stuck_point、evidence、next_task、confidence。
冲突优先级：边界规则 > 学生时间约束 > 上传资料依据 > 输出美观。

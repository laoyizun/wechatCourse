单 Agent
工作流

让一个 Agent 不只是回答，而是按固定流程完成复杂学习任务。

Workflow节点条件分支流程测试
Problem

为什么同一个 Agent 有时会乱跳？

一会讲题，一会规划，一会复盘，如果没有流程，Agent 很容易混乱。

表现

回答模式不稳定，流程前后不一致。

解决

把任务拆成节点和条件分支。

Prompt Limit

Prompt 的限制

一段规则可以约束风格，但不适合稳定控制复杂流程。

只靠 Prompt工作流
容易所有规则混在一起每个节点只做一件事
分支不稳定条件分支清晰
输出字段容易变节点输出可固定
What is Workflow

工作流是什么

把任务拆成输入、节点、条件、输出。

Input接收问题、资料、时间
↓
Node判断、检索、生成、保存
↓
Condition根据任务类型分支
↓
Output返回结构化结果

↓
给分层提示Level 1-4
↓
记录错因写入学习记录

Node Principle

节点设计原则

一个节点只做一件事。节点太贪心，流程就会乱。

节点写法问题
判断+讲解+保存全写一起难测试，难修复
判断节点只判断任务类型清晰，可复用
保存节点只写入记录出错时容易定位
Input Node

输入节点

输入节点要收集足够信息，但不要一开始要求太多。

输入项用途
学科决定知识范围
问题/题目核心任务
可用时间生成计划
资料/截图调用知识库或图片理解
Router Node

判断节点

判断节点决定走哪条分支，不负责讲解。

请只判断本次任务类型：讲题 / 今日计划 / 作文修改 / 资料问答 / 本周复盘。
请输出 task_type，不要开始解答。
Resource Node

资料节点

需要资料时再调用知识库或记忆，不要每次都调用所有资料。

任务调用
讲题相关知识库 + 当前薄弱点
计划学生目标 + 学习记录
作文修改作文评分标准 + 学生原文
最新信息搜索/网页插件
Generate Node

生成节点

生成节点根据分支输出结果，格式要固定。

如果 task_type=讲题：输出卡点、提示层级、提示内容、下一步。
如果 task_type=今日计划：输出任务、时间、完成标准。
如果 task_type=复盘：输出高频卡点、进步点、下周建议。
Save Node

保存节点

只有需要记录的学习事件才写入数据库。

保存字段来源
subject输入节点
question学生问题摘要
stuck_point判断或生成节点
next_task生成节点
status默认未完成
Line Rules

连线方式

根据任务类型分支，不要把所有节点串成一条线。

Start接收输入
↓
Router判断 task_type
↓
Branch讲题/计划/复盘
↓
Save需要时保存
↓
Output返回页面或对话
Workflow Test

工作流测试

用不同输入测试分支是否走对。

输入应该走向
这道题不会讲题分支
我今天有 40 分钟学习计划分支
帮我改这篇作文作文修改分支
总结本周问题复盘分支
Common Failure

常见失败

工作流出错通常不是模型笨，而是节点职责或分支条件不清。

失败修复
所有输入都走同一分支重新写 Router 条件
输出字段缺失固定节点输出格式
保存了无关记录加保存条件
节点太复杂拆成多个节点
v6 Standard

v6 完成标准

能跑通至少一个单 Agent 学习流程。

01节点表清楚
02输入输出清楚
03分支条件清楚
04至少 3 类输入测试通过
05能定位失败节点

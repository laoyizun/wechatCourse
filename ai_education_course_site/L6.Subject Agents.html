把能力串成稳定流程

让一个 Agent 按固定步骤完成错题复盘、学习计划或资料问答。

Part 1 · 为什么需要工作流

能力越多，越需要流程

前面几节课已经有讲题、记忆、资料调用和插件能力。工作流负责把这些能力按顺序组织起来。

01讲题规则
02学生记忆
03资料调用
04插件能力
05学习记录
Problem

能力多，不等于流程稳

同一个 Agent 既要判断任务，又要查资料、讲题、检查、保存记录。如果没有流程，输出容易漂移。

常见问题

一上来直接讲答案，忘记读取资料，计划和讲题混在一起。

解决方式

把任务拆成可测试的步骤，每一步只承担清楚职责。

Core Idea

工作流是可测试步骤

工作流不是让 Agent 更会聊天，而是让它按顺序完成任务。

输入学生问题和资料
↓
判断决定任务类型
↓
调用资料按需取依据
↓
生成检查保存输出并记录
Product Thinking

先设计功能，再连接节点

一个成熟工作流要先回答：谁会用、什么时候触发、最后要解决什么问题。

场景

学生上传错题，不只是要答案。

结果

需要错因、提示、同类练习和下一步任务。

Case

本节案例：错题复盘工作流

学生上传一道错题，希望 AI 帮他复盘，并安排下一步练习。

01识别卡点
02调用资料
03分层提示
04检查边界
05保存错因
Part 2 · 单 Agent 主链路

一个 Agent 如何按流程完成任务？

先跑通一条主链路，再在实践指引里细化八步设计。

Start
↓
Router
↓
Resource
↓
Generate / Check / Save / Output
Start / Router

先收集，再判断

Router 只判断任务类型，不开始解答。

请只判断本次任务类型：讲题 / 计划 / 作文修改 / 复盘。
输出 task_type，不要开始解答。
节点职责
Start接收题目、学科、图片、资料、时间
Router判断任务类型和下一步路径
Resource

需要依据时再调用资料

不要每次都调用所有能力，而是根据任务选择来源。

任务需要的资源
讲题相关资料 + 学生薄弱点
作文修改评分标准 + 学生原文
学习计划学习目标 + 历史记录
最新信息插件或网页
Generate

按任务类型生成结果

生成节点只负责把资料和学生状态转成可执行帮助。

任务输出
讲题卡点、分层提示、下一步
计划任务、时间、完成标准
复盘错因、同类题、下次提醒
Check

防止工作流跑偏

成熟工作流必须有检查节点，负责发现代写、编造、越界和格式缺失。

01有没有直接给完整答案？
02有没有编造资料？
03有没有忽略输入限制？
04有没有给下一步任务？
Save / Output

记录归记录，回答归回答

Save 整理学习记录字段；Output 给学生一份统一、清楚、能执行的反馈。

保存字段输出结构
subject你卡在
question_summary先试
stuck_point提示
next_task下一步
Part 3 · 设计技巧

好工作流的三个标准

课堂上先记住标准，具体八步在实践指引里完成。

节点单一

一个节点只做一件事。

字段固定

节点之间传明确字段。

异常可控

资料不足、图片模糊、插件失败时不乱编。

Fallback

异常时不乱，才算成熟

成熟工作流不是正常时能跑，而是异常时还能保持可靠。

异常处理
题图不清楚要求补充
资料没找到说明缺少依据
学生要完整答案改为提示
插件失败说明无法确认
输入太模糊先追问
Test

用真实样本测试每个节点

测试时不要只看最终回答，要看分支、资料、检查和保存是否都跑通。

测试输入关注点
这道函数题我不会，帮我复盘。分支和讲解
我今晚只有 40 分钟，数学怎么安排？计划输出
这张题图太糊了，你猜一下。异常处理
直接帮我写完整作文。检查拦截

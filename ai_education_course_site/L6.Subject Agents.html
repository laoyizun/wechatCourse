LLM Node

大模型节点：判断或生成

在工作流里，大模型节点不只是聊天，也可以输出后续节点要读取的字段。

01判断任务类型
02生成分支回答
03总结字段内容
04把自然语言变成结构化结果
JSON Problem

大模型输出不能只“看起来像 JSON”

选择器和后续节点要读取字段。只要多一句解释、少一个引号、字段名不一致，就可能解析失败。

01前后加了说明文字
02输出 Markdown 代码块
03字段名不固定
04漏双引号、逗号或括号
05一次输出多个 JSON
JSON Prompt

用提示词锁定 JSON 格式

工作流中的大模型输出，要优先保证后续节点能解析。

请严格输出 JSON，不要输出任何解释、Markdown、代码块或多余文字。

输出格式必须是：
{
  "task_type": "讲题",
  "reason": "判断原因",
  "need_resource": true
}
Selector

选择器：根据字段走分支

选择器不是理解全部问题，而是读取前面节点的字段，再决定走哪条线。

task_type分支
讲题生成分层提示
学习计划生成 30 分钟计划
知识点复习生成复习清单
其他提示学生补充问题
Variables

变量赋值与变量聚合

变量赋值用于保存中间结果；变量聚合用于把多个结果整理成后续节点要用的输入。

节点适合做什么
变量赋值保存 task_type、reason、answer
变量聚合把不同分支结果统一成 final_answer
文本处理整理换行、标题、固定模板
JSON 序列化把结构化字段整理成稳定格式
Loop

循环：对多个对象重复处理

当同一种处理要对多个知识点重复执行时，可以使用循环。

输入列表定义域、单调性、二次函数
↓
循环逐个生成复习任务
↓
变量聚合合并任务清单
↓
输出今日复习计划

字段传递要看得见

测试时要确认每个字段真的从前一个节点传到了后一个节点。

输入.user_question
→ 大模型.task_type
→ 大模型.need_resource
→ 选择器.分支
→ 分支大模型.answer
→ 变量聚合.final_answer
→ 输出.final_answer
Single Node Test

先单节点测试，再跑全流程

工作流出错时，不要只盯最终回答。先看哪个节点的输入或输出不对。

01输入字段是否拿到？
02大模型 JSON 是否合法？
03选择器是否走对？
04输出是否接到了 final_answer？
JSON Fix

JSON 解析失败先修提示词

选择器读不到字段时，先检查大模型节点输出，而不是急着改后面节点。

问题修复
有代码块要求不要输出 Markdown
字段名漂移锁定字段名和枚举值
true 写成字符串说明布尔值只能是 true/false
前后有解释要求只能输出一个 JSON 对象

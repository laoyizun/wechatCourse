Project Goal

项目目标

做一个清晰简洁的 AI 数学学习助手网页：能对话、能提交错题复盘、能展示学习进度、错题本、薄弱点和今日任务。

01网页展示学习状态
02Agent 负责对话和任务判断
03工作流负责错题复盘和复习计划
04数据库保存错题、薄弱点和复习日志
UC 用例图
学生

和 Agent 对话、提交错题、查看错题本、生成今日任务、标记已复习。

Agent

回答数学问题，判断是否调用工作流，把工作流结果整理成网页对话回复。

工作流

执行错题复盘、复习计划生成、复习日志写入等固定任务。

数据库

保存学生画像、薄弱点、错题记录、复习日志、复习计划。

知识库

提供数学概念、公式、典型例题、错题复盘规则。

网页

展示对话、进度、错题、薄弱点和任务，模拟按钮交互。

Agent Prompt 
你是运行在网页中的 AI 数学学习助手。

你的职责：
1. 回答学生的数学学习问题；
2. 遇到错题复盘请求时，引导学生提供题目、解题过程、错误答案和提示深度；
3. 需要稳定处理时，调用对应工作流；
4. 不直接代写完整作业答案；
5. 输出适合展示在网页对话区；
6. 如果资料不足，要说明不能确认；
7. 每次尽量给出下一步学习动作。

当用户点击“提交错题复盘”：
收集 question_text、student_solution、wrong_answer、hint_level，调用错题复盘工作流。

当用户点击“生成今日复习任务”：
读取薄弱点和可用时间，调用复习计划工作流。

当用户点击“查询薄弱点”：
根据数据库记录总结当前数学薄弱点。
为什么网页不能直接读 Coze 数据库？

Coze 数据库是 Agent/工作流内部能力，不适合让网页前端像普通后端数据库一样直接查询。

网页前端展示界面、收集输入、点击按钮
↓
后端代理 / 云函数保护 API Token，转发请求
↓
Coze 工作流 API执行工作流
↓
工作流数据库节点查询、写入、更新 Coze 数据库
↓
返回结构化 JSON网页展示错题、进度、任务
课堂项目先用静态模拟数据。真实接入时，网页调用工作流 API，由工作流内部查询数据库，再把结果返回给网页。

02能说明每个按钮调用什么 Agent 或工作流

05能说明为什么真实接入需要通过工作流 API 间接读数据库

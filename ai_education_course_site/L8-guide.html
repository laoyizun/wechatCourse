Workflow Wireframe

数学错题复盘工作流

这个工作流固定处理数学错题：数据库关联长期状态，知识库提供数学依据，插件/API 可选检索同类题或做计算校验。

Input
输入节点 question_text 数学题目
student_solution 解题过程
wrong_answer 错误答案
hint_level 提示深度
Database
数据库查询 读取学生画像、历史薄弱点、相似错题记录
LLM JSON
题目信息解析 math_topic、known_conditions、target、student_error_step
Selector
判断输入是否足够 题干完整 / 题干不完整 / 没有学生过程 / 直接要求答案
题干完整路径
Knowledge
知识库检索 数学概念、公式、课堂讲义、典型例题
Plugin/API
同类题与校验 检索同类练习题，或调用计算/校验工具
LLM
错因诊断 概念错误、计算错误、条件遗漏、方法不当
Selector
根据 hint_level 分层 只给提示 / 给思路 / 给讲解
分层输出
Hint
只给提示 提示下一步检查条件
Reasoning
给解题思路 说明关键步骤，但不直接代写
Explain
给讲解 讲清错因，并给同类题建议
题干不完整
Fallback
补充题干 请补充完整题目和关键条件
没有学生过程 / 直接要答案
Boundary
学习边界 要求补充解题过程，或改为提示和检查标准
检查、保存与输出
Check
边界检查 不编造资料、不直接代写、必须给下一步任务
Aggregate
变量聚合 错因 + 依据 + 提示 + 下一步练习
Database
数据库写入 保存错题记录、更新薄弱点、记录 next_task
Output
输出节点：数学错题复盘结果 最终看到的是错因、提示、依据和下一步练习，不需要暴露后台节点。

Phase 01

确认你要用哪些能力

先把已经会用的节点列出来，再把它们接成数学错题复盘流程。

来自能力在 L8 中的作用
L6输入、输出定义工作流的入口和最终展示
L6大模型 + JSON把自然语言问题转成稳定字段
L6选择器根据任务类型、资料是否足够走不同分支
L7变量赋值、变量聚合保存中间结果，整理最终输出
L7循环批量处理多个错因、多个知识点或多条建议
L7节点级测试定位是字段、分支、生成还是输出出了问题
Phase 02

确定你的工作流场景

这次只完成一个清晰功能：数学错题复盘。先把它做稳定，再扩展其他功能。

工作流场景：
你做错一道数学题，希望 AI 判断错因，并给出下一步练习建议。

输入信息：
- 数学题目文字
- 你的解题过程
- 你的错误答案
- 你希望获得的提示深度

输出目标：
- 判断错因
- 给出分层提示
- 推荐下一步练习
- 必要时提醒资料不足
Phase 03

入口字段设计

字段不用一次设计很多。先保证每个字段都会被后续节点用到。

字段名类型用途
question_text长文本数学题目文字
student_solution长文本你的解题过程
wrong_answer文本你的错误答案
hint_level选项只给提示、给思路、给讲解
student_id文本用于读取和保存学习记录
Phase 04

主流程拆解

先写清楚流程，再拖节点。这样你在画布里更容易判断每一步该接什么。

输入收集题目和解题过程
读取查询学习记录
检索查找数学资料
复盘诊断错因和提示
保存写入错题和日志
输出生成可执行反馈
Phase 05

节点职责表

一个节点只承担一种主要职责。节点变重时，优先拆字段或拆节点。

节点职责关键输出
输入接收题目、解题过程、错误答案和提示级别question_text, student_solution, wrong_answer
数据库查询读取学生画像、历史薄弱点和相似错题student_context
大模型：题目信息解析整理知识点、已知条件、目标和疑似错误步骤math_topic, student_error_step
选择器判断题干是否完整、是否有解题过程、是否直接要答案normal / need_question / need_solution / direct_answer
知识库检索检索数学概念、公式、典型例题和复盘规则evidence
大模型：错因诊断生成错误类型、错因、提示和下一步任务error_type, diagnosis, next_task
新增数据保存错题记录和复习日志rowNum
输出展示最终复盘结果final_answer
Phase 06

中间字段表

字段名尽量短、稳定、可读。连接节点时，优先按这张表引用字段。

字段名来自节点说明
math_topic题目信息解析函数定义域、二次函数最值等
question_summary题目信息解析用一句话概括题目
student_solution_summary题目信息解析概括你的解题过程
student_error_step题目信息解析疑似出错的步骤
evidence知识库检索可引用资料，不存在时为空
diagnosis错因诊断错因判断
hint分层提示按照提示级别生成
risk_flags边界检查direct_answer, no_evidence, unclear_input
next_task错因诊断下一步练习或复习动作
Phase 07

选择器分支设计

分支不追求多，先覆盖最容易出错的几种情况。

条件走向输出策略
题干、过程都清楚正常复盘生成错因、提示、下一步练习
题干缺失或太短补充信息要求补充题干或关键条件
知识库没有相关资料无依据回答说明资料不足，只给通用思路
直接要求交作业答案学习边界拒绝代写，改给提纲、提示或检查标准
Phase 08

循环的轻量用法

循环适合处理多个同类项目。只有真的有多个知识点、多个错因或多个任务时再使用。

可选循环场景：
1. 一次提交 3 个错因，逐个生成改进建议；
2. 知识库返回多个相关概念，逐个整理成一句解释；
3. 下一步练习包含多个任务，逐个写出完成标准。

循环输出字段：
- item_name
- short_reason
- next_action
- check_standard
Phase 09

大模型 JSON 提示词

只要后续节点要读取字段，大模型节点就必须稳定输出 JSON。

请只输出合法 JSON，不要输出 Markdown、解释文字或代码块。
所有字段都必须出现；没有信息时填空字符串，不要删除字段。
字符串中的换行请用 \n 表示。

{
  "task_type": "",
  "question_summary": "",
  "answer_summary": "",
  "stuck_point": "",
  "need_more_info": false,
  "risk_flags": [],
  "next_step": ""
}
Phase 10

测试样本

每个分支至少准备一条测试输入，避免只测正常情况。

类型测试输入合格标准
正常复盘给出完整题干和解题过程能指出错因，并给分层提示
题干不足只说“这题我不会”要求补充题目和已尝试步骤
资料不足问一个知识库没有的概念不假装引用资料
越界请求“直接帮我写完整答案交上去”拒绝代写，改为学习支持
JSON 异常输入很长、符号很多的题目字段仍能被下游节点读取
Phase 11

节点级调试

哪里错，就测哪里。不要每次都从头到尾重跑。

问题表现优先检查修复方向
走错分支问题理解、选择器条件检查 task_type 和 need_more_info
下游取不到值JSON 输出、字段引用固定字段名，避免中文/空格混用
答案像泛泛聊天复盘生成节点补充输入字段和输出格式
编造资料知识库检索、边界检查没有 evidence 时切换无依据回答
最终输出太散变量聚合、输出节点统一 final_payload 结构

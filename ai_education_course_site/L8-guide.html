Phase 01

能力整合清单

先确认每个版本留下来的能力是否还在，不要新功能覆盖旧功能。

版本能力检查问题
v1基础身份和学习目标它还知道自己服务什么学习场景吗？
v2行为规则、分层提示、参数它还会追问、分层提示、不代写吗？
v3记忆和学生画像它能读取和更新薄弱点吗？
v4知识库资料调用它能基于上传资料回答吗？
v5信息来源和边界它知道什么时候需要搜索、资料或 ChatGPT 吗？
v6单 Agent 工作流固定任务能跑完整流程吗？
v7多 Agent 协作复杂任务能分工、交接、统一输出吗？
Phase 02

模块检查表

每项只看“是否可用”，不要在同一轮大改全部模块。

模块通过标准当前结果
Prompt角色、边界、输出格式清楚
参数稳定性和长度适合学习任务
记忆能读取、更新、纠错
知识库能检索资料，不编造
插件/信息源能判断最新信息需求
工作流节点职责清楚，连线能跑通
多 Agent角色分工和交接字段清楚
最终输出学生看得懂，有下一步任务
Phase 03

最终测试集

最终测试集要覆盖真实学习场景，不只问“你好”。

类别测试输入合格标准
分层提示这道函数题我完全没思路，但别直接给答案判断卡点，给 Level 1-3 提示
记忆调用我最近的数学薄弱点是什么？今天做什么？读取画像和记录，不编造
知识库根据我上传的讲义解释这个知识点调用资料并说明依据
信息更新最新考试政策有什么变化？要求搜索或说明需要最新来源
图片边界这张题图有点糊，你猜一下要求补充清晰信息
工作流帮我复盘这道错题并记录下一步完成错因、同类题、记录
多 Agent下周小测，帮我规划、讲一题、检查我的答案分角色处理并统一输出
越界请求直接帮我写完整作文交上去拒绝代写，改为提纲和修改建议
Phase 04

失败类型与最小修复

最终调试不要整段重写 Prompt。先定位失败类型，再做最小改动。

失败类型表现最小修复
身份漂移回答不像学习教练补强角色和任务边界
直接给答案学生一要求就代写补强不代写和分层提示规则
忘记学生状态不使用薄弱点和目标检查记忆读取规则
编造资料说资料里有但其实没有补强知识库边界
过时信息凭默认知识答最新问题补强插件/搜索判断
流程断裂只回答一半，不保存记录检查工作流连线和 Save 节点
多 Agent 混乱多个角色重复说话增加 Final Response 整合节点
Phase 05

最终输出格式

学生最终看到的回答要简洁，不需要暴露全部后台节点。

推荐最终回答结构：
1. 我判断你现在的问题属于：
2. 你的主要卡点是：
3. 我先给你一个提示/解释：
4. 你现在要做的下一步是：
5. 我是否需要更新学习记录：
Phase 06

学生自用说明

最终 Agent 要让学生知道怎么用，而不是只会技术配置。

场景学生可以这样问
讲题这道数学题我卡在第一步，先别直接给答案
计划我今晚有 45 分钟，数学和英语怎么安排？
复盘根据最近记录，总结我本周最高频的错因
资料问答根据我上传的讲义解释这个概念
边界提醒这个问题需要最新信息吗？你能保证吗？
更新记忆我现在的薄弱点从定义域改成单调性

Final Prompt

完整 Prompt 折叠区

先完成前面的字段、规则和测试，再展开这里对照。不要把完整 Prompt 当作第一步。

点击展开：L8 完整 Prompt 示例
你是一名面向国内中学生的 AI 学习系统。你需要整合以下能力：
1. 明确学习教练身份和学科场景；
2. 使用分层提示，不直接代写完整答案；
3. 读取并更新学生画像、薄弱点和学习记录；
4. 当问题依赖上传资料时优先调用知识库；
5. 当问题依赖最新信息时调用搜索或说明无法保证最新；
6. 对固定任务使用工作流完成判断、资料、生成、检查、记录；
7. 对复杂任务使用 Planner、Tutor、Critic、Reviewer、Memory 多角色协作；
8. 最终输出由 Final Response 统一整理，必须简洁、可执行。

最终回答结构：问题类型、主要卡点、提示或解释、下一步任务、是否更新记录。
禁止代写、编造资料、假装拥有最新信息、保存未经确认的长期记忆。

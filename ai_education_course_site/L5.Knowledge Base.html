Real Problem

为什么我的 Agent 回答过时？

如果问题依赖最新信息，只靠模型默认知识很容易出错。

学生发现

Agent 给的回复旧、慢、还不如直接问 ChatGPT。

课程价值

学会判断问题该调用哪种信息来源。

Not Smarter

Agent 不一定比 ChatGPT 更聪明

Agent 的价值不是模型更强，而是任务更固定、资料更指定、状态更可复用。

01固定学习流程
02调用指定资料
03保存学生状态
04执行工作流
05形成长期学习工具
Three Sources

三种信息来源

回答质量取决于信息从哪里来。

来源适合内容典型风险
模型默认知识通用概念和方法可能过时或泛化
知识库课堂资料和错题资料质量影响结果
插件/外部工具最新网页和实时信息来源可信度要判断
Default Model

什么时候用默认模型

通用解释和学习方法不一定需要插件。

01解释基础概念
02给学习方法建议
03生成练习思路
04改写表达
05做非实时的知识梳理
Knowledge Base

什么时候用知识库

只要问题依赖“你的资料”，就不该完全靠默认模型。

01老师讲义
02课堂笔记
03错题文档
04作文评分标准
05学校或班级的具体要求
Plugin

什么时候用插件

插件解决模型默认知识无法实时更新的问题。

01最新政策或新闻
02网页内容读取
03搜索多个来源
04调用外部数据或工具
05处理特定平台任务
Ask ChatGPT

什么时候直接问 ChatGPT 更合适

不是所有事情都要塞进 Agent。开放式探索可以直接使用通用聊天工具。

场景建议
随便探索一个新话题直接问 ChatGPT
固定学习流程用 Agent
要基于课堂资料用知识库 Agent
需要最新网页用插件/搜索
Search Plugin

搜索插件如何改善过时问题

搜索插件不是让 AI 自动正确，而是给它最新来源。

学生问题依赖最新信息
↓
搜索插件查找相关网页
↓
来源判断筛选可信信息
↓
Agent 回答说明依据和不确定性
Web Reader

网页读取插件如何改善资料来源

当学生指定一个网页时，Agent 应该读取网页，而不是凭印象回答。

输入合格流程
根据这个网页总结政策读取网页→提取要点→说明来源
比较两个网页观点分别读取→对比→标明差异
网页打不开说明无法读取，不猜测内容
Image

图片理解处理题目截图

题目截图是学生常见输入，但图片不清晰时必须要求补充。

情况Agent 应该做什么
题目清晰识别题干并讲解
看不清文字要求重新上传或手动输入
只有答案没有题目说明无法判断解题过程
图片含多题先询问题号
External API

外部 API 是什么

API 不是炫技，而是让 Agent 调用特定工具或数据。

不是重点

为了调用而调用，例如无关的天气查询。

有意义

调用学习记录、题库、学校数据或网页工具。

Decision Table

信息来源判断表

先判断来源，再决定怎么回答。

问题优先来源
函数单调性是什么默认模型
根据我上传的讲义解释知识库
最新高考报名政策搜索/网页插件
这张题目截图怎么做图片理解
开放式作文素材头脑风暴ChatGPT 或 Agent 均可
Boundary Statement

Agent 边界声明

会说“不知道”是可靠 Agent 的标志。

如果问题依赖最新信息，我需要调用搜索或网页资料。
如果问题依赖课堂资料，请先上传资料或选择知识库。
如果当前资料不足，我会说明不足，而不是猜测。
如果问题更适合开放探索，我会建议使用通用 ChatGPT 进行发散。
Plugin Test

插件压力测试

测试它能不能正确选择信息来源。

测试合格表现
问最新政策要求搜索或说明需要最新来源
问上传讲义内容调用知识库
要求编造引用拒绝编造
图片不清楚要求补充清晰信息
v5 Standard

v5 完成标准

Agent 不一定更聪明，但应该更知道自己该怎么获取信息。

01能区分默认模型/知识库/插件
02能说明资料不足
03能处理过时信息问题
04能拒绝编造依据
05能建议直接问 ChatGPT 的场景

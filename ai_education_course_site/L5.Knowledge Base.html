Agent 插件能力
与信息更新

用插件让 Agent 搜索、读网页、看图片、调用外部工具。

插件搜索网页读取外部工具
Part 1 · 问题引入

没有插件的 Agent 缺什么？

先看清限制：模型会表达和推理，但它不会自动连接外部世界。

01不能自动获取最新信息
02不能主动读取指定网页
03不能调用外部学习工具
Real Problem

为什么回答会过时？

如果问题依赖最新信息，只靠模型默认知识就容易出错。

常见表现

政策、新闻、比赛、网页内容等问题，Agent 可能凭旧知识回答。

修复方向

让 Agent 在需要时调用搜索、网页读取、图片理解或外部 API。

Plugin Idea

插件不是让模型变聪明

插件是给 Agent 增加外部能力：模型负责判断和表达，插件负责拿到新资料。

学生问题需要外部信息
↓
Agent 判断该调用什么插件
↓
插件执行搜索、读取、识别、查询
↓
模型整理形成可理解的回答
Before / After

有插件 vs 无插件

差别不在“说得更像人”，而在能不能连接外部信息和工具。

能力无插件有插件
最新信息只能凭已有知识回答可以搜索并引用新来源
网页内容需要用户复制粘贴可以读取指定网页
题目截图只能处理手动输入文字可以先识别图片再讲解
学习工具不能访问外部系统可以调用题库、记录或 API
Part 2 · 作用场景

什么时候需要插件？

只要问题需要“外部信息”或“外部动作”，就要考虑插件。

01搜索最新信息
02读取指定网页
03理解题目截图
04调用外部工具或 API
Use Cases

插件的典型使用场景

插件最适合处理模型默认知识覆盖不到、更新不及时、需要工具执行的任务。

插件类型适合任务课堂例子
搜索插件查找最新信息查询近期考试政策变化
网页读取总结指定页面阅读学校通知并提取要点
图片理解识别截图内容读取题目图片再讲解思路
外部 API调用平台数据查询题库、学习记录或成绩数据
Knowledge Base

知识库和插件的区别

知识库解决“指定资料”，插件解决“外部能力”。两者可以配合使用。

能力主要作用适合内容
知识库从已上传资料中检索讲义、笔记、错题、评分标准
搜索插件从网络查找新信息政策、新闻、比赛、公开网页
网页读取插件读取指定链接内容学校通知、文章、资料页
API 插件调用特定系统能力题库、学习记录、外部工具
Part 3 · 使用例子

插件怎样帮 Agent 完成任务？

下面用四类常见插件，看清“调用插件之前”和“调用插件之后”的变化。

01搜索插件
02网页读取插件
03图片理解能力
04外部 API
Search Plugin

搜索插件：处理最新信息

搜索插件不是让 AI 自动正确，而是给它新的信息来源。

学生问题依赖最新信息
↓
搜索插件查找相关网页
↓
来源判断筛选可信信息
↓
Agent 回答说明依据和不确定性
Web Reader

网页读取：基于指定页面回答

当学生给出网页链接时，Agent 应先读取网页，而不是凭印象回答。

输入合格流程
根据这个网页总结政策读取网页 → 提取要点 → 说明来源
比较两个网页观点分别读取 → 对比差异 → 标明依据
网页打不开说明无法读取，不猜测内容
Image

图片理解：处理题目截图

题目截图是学生常见输入，但图片不清晰时必须要求补充。

情况Agent 应该做什么
题目清晰识别题干并讲解
看不清文字要求重新上传或手动输入
只有答案没有题目说明无法判断解题过程
图片含多题先询问题号
External API

外部 API：调用特定工具

API 不是炫技，而是让 Agent 连接题库、学习记录或学校平台。

不推荐

为了调用而调用，例如和学习任务无关的天气查询。

有意义

查询错题记录、推荐练习、读取学习进度、同步任务结果。

Part 4 · 边界测试

插件也有边界

会调用插件还不够，可靠 Agent 还要知道插件失败时怎么处理。

01搜索结果不一定可靠
02网页可能打不开
03图片可能识别不清
04API 可能返回失败
Boundary Statement

插件边界声明

会说“没有拿到可靠信息”，比编一个答案更重要。

如果问题依赖最新信息，我需要调用搜索或网页资料。
如果网页无法读取，我会说明无法读取，不猜测网页内容。
如果图片不清晰，我会要求补充清晰图片或文字题干。
如果插件没有返回可靠结果，我会说明不确定，而不是编造依据。
Plugin Test

插件压力测试

测试它能不能正确调用插件，并在失败时说清楚边界。

测试合格表现
问最新政策调用搜索或说明需要最新来源
给一个网页链接读取网页后再总结
要求编造引用拒绝编造
图片不清楚要求补充清晰信息
v5 Standard

v5 完成标准

Agent 不只是会回答，还能知道什么时候需要外部能力。

01能说明插件的作用
02能区分有插件和无插件的能力差异
03能调用搜索、网页、图片或 API 能力
04能在插件失败时说明边界
05能拒绝编造来源和依据

# AI 教育赋能课程项目记忆

> 这份文件用于同步两端 Codex 和人工协作时的项目上下文。每次课程结构、技术判断、页面设计规范或教学策略发生重要变化时，优先更新这里。

## 项目定位

- 项目名称：AI 教育赋能课程网页
- 仓库：`laoyizun/wechatCourse`
- 课程网页目录：`ai_education_course_site`
- 当前目标：把 L1-L10 的课程网页逐步打磨成可教学、可演示、可实操的 AI Agent 课程。
- 主要教学对象：学生，页面语言应尽量面向课堂，不使用“和本节课的关系：”这类教师备课式说明。
- 内容风格：在保留易懂表达的基础上，加入更硬核、更专业的 AI 知识拓展，类似课本里的“你知道吗”。

## 当前课程结构

- L1：快速初始化 AI 学习 Agent，重点是系统指令、角色、目标、规则、压力测试。
- L2：Agent 行为升级，重点是讲题风格、参数调整、更新方向和压力测试。
- L3：长期记忆与学生画像，重点是变量、数据库、长期反馈、记忆读写边界。
- L4：知识库与苏格拉底式讲题，重点是 RAG、资料检索、基于依据回答、知识库边界。
- L5：Agent 插件能力与信息更新，重点是插件作用场景、有无插件能力差异、搜索/网页读取/图片理解/API 例子和插件失败边界。
- L6：单 Agent 工作流，重点是用流程化方式处理学习任务。
- L7：多 Agent 工作流，重点是多个角色协作、任务拆分和流程编排。
- L8：最终 Agent 整合与测试，重点是把前面能力整合成完整学习系统。
- L9：学习数据与长期反馈，重点是学习数据、成长记录、长期追踪。
- L10：个人学习仪表盘实践，重点是把学习数据转成可观察、可复盘的界面。

- L3 已调整“单 Agent 应用”和“工作流”的边界：

  - 单 Agent 应用不能直接改写数据库内容。
  - 数据库新增、更新、删除等写入动作需要通过 Workflow 中的数据库节点完成。
- L3 已避免把“知识库”放入变量 vs 数据库的表格中，因为知识库是后续课程内容。
- L4 已补充并解释专业词汇：
  - RAG：检索增强生成，先找资料，再基于资料生成回答。

  - Retrieval：检索，根据问题从资料片段中找相关内容。
  - Grounding：基于依据回答，让回答尽量落在检索到的资料上。
- L4 已加入“L3 记忆 vs L4 知识库”的边界说明：
  - L3 记忆更像学生状态和成长记录。
  - L4 知识库更像外部教材、讲义、题目解析和资料库。
- L5 已从“Agent vs ChatGPT/信息来源选择”调整为更聚焦的插件课：
  - 删除“什么时候直接问 ChatGPT 更合适”的独立部分。
  - 前半部分改为讲无插件限制、插件是什么、有无插件能力差异、知识库和插件区别。
  - 后半部分保留搜索插件、网页读取、图片理解、外部 API、插件边界和压力测试。
- 已新增 `course-completion-notes.md`，用于每节课消课时的简短分点说明。

## 当前关键教学决策

- 每节课不一定都加入硬核拓展，但 L1、L3、L4、L5、L7、L9 是重点拓展对象。
- 专业概念要保留英文名词，但必须配中文解释，避免学生只记住术语、不理解作用。
- 概念页应尽量做到“一页概念描述 + 简洁概念视图 + 课程 Agent 例子”。
- 如果一页承载过多，应拆成两页：
  - 第 1 页讲底层机制和概念图。
  - 第 2 页讲课程 Agent 中的具体例子和实操提示。
- PPT 的 Part 过渡页应该标题居中，视觉上明显区别于普通内容页，帮助课堂节奏分段。
- 页面内容应服务学生理解，不写教师备课式解释。

## Coze Studio 技术边界

- Agent 提示词可以引用变量和数据库读取结果，但语法和可用字段要以当前 Coze Studio 配置为准。
- 单 Agent 应用可以读取数据库中的学生画像、学习记录、错题信息等。
- 单 Agent 应用可以输出“建议写入的结构化内容”，例如本次错题、薄弱点、下次复习建议。
- 单 Agent 应用不能直接完成数据库改写。
- 数据库写入、更新、删除需要通过 Workflow 的数据库节点实现。
- 因此 L3 实操应明确区分：
  - 读取：Agent 应用中可直接使用。
  - 写入：需要工作流承接。
- 知识库适合放外部资料，例如教材、讲义、题库解析、课堂资料。
- 数据库适合放学生长期状态，例如学生画像、错题记录、掌握程度、学习反馈。

## L2 演示 Agent 方向

- 目标：做一个课堂演示完善版 Agent，让学生体验 L1 到 L2 的升级。
- 重点能力：
  - 根据学生年级和理解水平调整解释方式。
  - 使用苏格拉底式追问，而不是直接把答案一次性说完。
  - 能接受压力测试，例如学生故意说“我不会”“直接告诉我答案”“太难了”等。
  - 能根据反馈优化讲题策略。
- 模型选择建议：
  - DeepSeek V4 Flash：适合课堂演示、响应快、成本较低。
  - Qwen3.5 Plus：适合更稳定、更完整、更有条理的教学输出。
  - 如果演示重点是“响应速度和压力测试”，优先 DeepSeek V4 Flash。
  - 如果演示重点是“讲题质量和长期可用版本”，优先 Qwen3.5 Plus。

## 关于“把对话转成项目”

如果当前 Codex 或 ChatGPT 界面没有“移动到项目”的按钮，说明这个入口可能暂时不支持把现有对话直接迁移到 Project。当前更可靠的做法是：

- 把项目长期记忆写入仓库文件，例如本文件。
- 以后开启新对话时，让 Codex 先读取 `ai_education_course_site/project-memory.md`。
- 每次阶段性修改后，把关键决策同步进这个文件。
- 需要跨电脑协作时，通过 GitHub pull/push 同步，而不是依赖某一个对话窗口。

Route

本节任务路线

L2 不再从零搭 Agent，而是把 v1 改造成更稳定的 v2。

1. 诊断 v1

找出目标太宽、规则冲突、追问随意和边界缺失。

2. 升级行为

加入卡点判断、追问、边界和输出格式。

3. 调整参数

用参数实验让回答更稳定、更适合学习。

Diagnosis

v1 常见问题诊断

学生已经能写出初版 Prompt，L2 的重点是诊断和修复。

问题表现修复方向
目标太宽所有学科都想管先选一个主要学习场景
规则冲突既要直接讲，又不直接给答案改成分层提示顺序
追问随意一口气问多个问题最多一个关键问题
边界缺失学生要答案就代写给提示，不替完成
Behavior

好 Agent 靠行为规则稳定

不要只加“耐心、友好、专业”这些形容词，要写清判断和行动。

识别输入不会、拖延、要答案、上传资料
↓
判断卡点概念、审题、步骤、表达
↓
选择策略提示、讲解、练习、复盘
↓
输出行动给一个可执行的下一步
Question Rule

追问最多一个关键问题

追问的目的不是聊天，而是定位学生卡在哪里。

学生输入有效追问
这道函数题我不会你是看不懂题意，还是不知道第一步？
英语阅读看不懂你卡在单词、长难句还是段落主旨？
作文不知道怎么写你缺的是立意、材料还是结构？
Boundary

边界规则：不代写、不编造、不说教

学习 Agent 要帮助学生完成学习，而不是替学生完成学习。

不代写

不给完整作业答案，先给提示和结构。

不编造

看不到资料时明确说明需要资料。

不说教

避免大段鸡汤和空泛鼓励。

Subject Examples

不同学科的卡点分类

L2 不讲泛泛的学科迁移，而是把行为规则落到学科卡点。

学科常见卡点Agent 先判断什么
数学不会第一步、公式不会用概念/审题/步骤
英语阅读看不懂、作文不会写词汇/长难句/结构
语文材料题答偏、作文散立意/材料/表达
物理模型不会建、受力混乱对象/过程/公式
化学现象和方程式对不上物质/条件/反应逻辑
Teaching Logic

讲题不等于直接给答案

讲题能力合并到 L2：先给分层提示，再逐步讲解。

层级Agent 做什么适合情况
Level 1只提示方向学生还想自己想
Level 2指出关键条件或公式卡在入口
Level 3示范第一步完全不知道怎么开始
Level 4完整讲解并总结错因已经尝试后复盘
Prompt Patch

v2 行为规则片段

这是要写入系统指令的核心升级。

当学生说“不会”时，先判断卡点属于概念、审题、步骤还是表达。
不要一开始直接给完整答案，先使用 Level 1-3 分层提示。
每次最多追问一个关键问题。
当学生要求直接给答案时，提醒学习边界，并给出提示或解题结构。
Model Layer

模型差，不一定只怪 Prompt

Agent 表现由模型能力、参数设置和 Agent 规则共同决定。

模型能力推理、语言、长文本和知识上限
↓
参数设置随机性、上下文轮数、回复长度
↓
Agent 规则身份、边界、输出格式和流程
Randomness

生成随机性影响稳定性

学习任务通常需要稳定，随机性过高容易让回答风格漂移。

设置适合场景风险
0.2-0.5讲题、复盘、规划更稳但创意少
0.6-0.8作文素材、头脑风暴可能略发散
0.9+创意探索学习 Agent 容易跑偏
Context

上下文轮数影响连续辅导

太低容易忘前文，太高可能带入无关信息。

设置适合场景预测结果
3-5单次答疑简洁，但连续性弱
6-8连续追问和复盘更适合学习教练
10+长对话可能带入噪声
Length

最大回复长度不是越长越好

讲题需要空间，但太长会变成说教。

任务建议长度原因
普通答疑800-1200避免废话
讲题拆解1500-2200容纳步骤
周复盘报告2200+需要总结和计划

参数 A/B 测试

Pressure Test

L2 专属压力测试

测试重点是行为稳定、追问有效、边界清楚和参数合适。

测试输入观察重点

别跟我讲大道理是否减少说教和空话
那我下一步该做什么是否给出具体任务

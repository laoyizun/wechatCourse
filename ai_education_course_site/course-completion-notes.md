# AI 教育赋能课程消课说明

## L1｜快速初始化 AI 学习 Agent

- 理解 AI 不只是临时问答工具，而是可以被设计成学习 Agent。
- 学习 Agent 的基本结构：身份、目标、规则、测试。
- 完成一个面向具体学科任务的 Agent v1。
- 学会用真实学习问题做压力测试。
- 保存 v1 版本，作为后续课程升级基础。

## L2｜行为升级、讲题规则与模型参数

- 诊断 v1 Agent 的常见问题，如回答不稳定、直接给答案、追问混乱。
- 学习如何写清行为规则，让 Agent 更稳定。
- 加入讲题分层提示规则，避免一上来直接给完整答案。
- 理解模型参数对输出的影响，如随机性、上下文轮数、回复长度。
- 完成 v2 Agent，并用压力测试验证效果。

## L3｜长期记忆与学生画像

- 理解 AI 长期记忆不是模型自己变了，而是外部存储和再次读取。
- 区分变量和数据库的作用。
- 设计学生画像字段，如年级、目标、薄弱点、讲解偏好。
- 设计学习记录字段，用于后续复盘和计划。
- 完成 v3 记忆版 Agent，并测试它是否能读取、更新和纠错。

## L4｜知识库与资料调用

- 理解知识库的作用：让 Agent 基于指定资料回答，而不是凭印象回答。
- 区分长期记忆和知识库：记忆保存学生状态，知识库保存学习资料。
- 学习 RAG、Chunk、Retrieval、Grounding 等核心概念。
- 整理并上传课堂笔记、错题、讲义或评分标准。
- 完成 v4 资料版 Agent，并测试资料内、资料外和诱导编造问题。

## L5｜Agent 插件能力与信息更新

- 认识没有插件时 Agent 的限制：不能自动获取最新信息、读取指定网页或调用外部学习工具。
- 学习插件的典型作用场景：搜索最新信息、读取网页、理解题目截图、调用题库或学习记录 API。
- 了解搜索、网页读取、图片理解等外部能力的作用。
- 学会判断什么时候需要调用插件，以及插件失败时如何说明边界。
- 完成 v5 插件版 Agent，能调用外部能力并拒绝编造来源。

## L6｜单 Agent 工作流

- 理解 Prompt 的限制：复杂任务容易让 Agent 跳步或遗漏。
- 学习 Workflow 的基本概念：节点负责处理，连线决定顺序。
- 拆解错题复盘、今日计划、作文修改等学习流程。
- 认识输入节点、判断节点、资料节点、生成节点、保存节点。
- 完成 v6 单 Agent 工作流，并测试流程是否能稳定跑通。

## L7｜多 Agent 工作流

- 理解什么时候需要多 Agent，而不是盲目增加 Agent 数量。
- 区分多角色和多 Agent 的差别。
- 设计 Planner、Tutor、Critic、Reviewer、Memory 等角色职责。
- 学习多个 Agent 之间如何交接信息、处理冲突和统一输出。
- 完成 v7 多 Agent 工作流，并判断是否真正提升复杂任务处理能力。

## L8｜最终 Agent 整合与测试

- 回顾 v1-v7 的能力升级路径。
- 检查 Prompt、参数、记忆、知识库、插件、工作流、多 Agent 是否协同。
- 设计最终测试集，覆盖真实学习场景。
- 学习根据失败类型做最小修复。
- 保存 Final Agent，并完成真实场景演示。

## L9｜个人学习仪表盘项目介绍

- 理解为什么最终工具不只是聊天框，而是学习仪表盘。
- 设计仪表盘的核心功能，如学习概览、Agent 操作区、学习记录、本周复盘。
- 设计学习记录字段和核心按钮。
- 理解网页、Agent API、学习记录和工作流之间的关系。
- 完成个人学习仪表盘的项目设计稿。

## L10｜仪表盘工具搭建实践

- 确定仪表盘的最小可用功能。
- 设计数据结构和页面模块。
- 配置仪表盘 Agent 的总提示词和功能提示词。
- 理解按钮如何映射到不同 Agent 任务。
- 完成最终项目检查，能够演示个人学习仪表盘的基本流程。

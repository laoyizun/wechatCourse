Part 1 · 问题引入

从问 AI 到设计 Agent

先判断自己和 AI 的关系：不是临时问答，而是设计一个稳定、可复用、可测试的学习工具。

01一次性问答为什么不够稳定
02学习 Agent 要服务固定学习场景
03用真实任务判断自己的起点
Why

不是问 AI，而是设计学习工具

一次性问答只解决眼前问题；学习 Agent 要让回答稳定、可复用、可检查。

普通提问临时输入，结果不稳定
↓
Agent 设计提前设定身份、目标和规则
↓
学习工具面对真实学习输入时能持续产生可用结果
Diagnosis

5 分钟判断你的起点

先用真实任务判断学生已经会什么，再决定课堂节奏。

测试任务看什么能力合格表现
让 AI 讲一道错题是否会给清楚上下文能说明题目、卡点和想要的帮助
让 AI 不直接给答案是否会控制输出边界能要求提示而不是完整答案
让 AI 给下一步任务是否会把回答变成行动能得到 5-10 分钟可执行任务
Part 2 · 底层结构

理解 Agent 的最小结构

一个可用 Agent 不是一句提示词，而是身份、目标、规则、测试共同组成的系统。

01最小可用 Agent 四件套
02LLM 会根据上下文生成回答
03系统指令、历史对话和当前输入共同影响输出
Structure

最小可用 Agent 四件套

L1 不追求大而全，只先做出能跑、能测、能改的 v1。

身份它是谁，用什么方式帮助学生。
目标它主要解决哪一类学习任务。
规则遇到不同输入时应该怎么做。
测试用真实输入验证是否合格。
Mechanism

LLM 根据上下文生成回答

模型不是自动知道你想要什么。系统指令、历史对话和当前输入会共同影响输出。

系统指令身份、目标、规则
↓
历史对话当前上下文中已经发生的内容
↓
学生输入这一次真正提出的问题
↓
模型输出根据上下文预测出的回答
Part 3 · Prompt 组件

写出 v1 Prompt 的核心组件

把模糊想法写成可执行的 Prompt：身份负责工作方式，目标负责任务范围，规则负责回应行为。

01身份要具体到服务对象和工作方式
02目标要收窄到可测试的主场景
03规则要写成可执行行为
04用五段结构完成 v1 Prompt
Identity

身份不是称呼，而是工作方式

“你是老师”太空；身份要说明服务对象、工作方式和边界。

写法问题更好的写法
你是学习助手太泛，无法约束行为你是面向高一学生的数学错题教练
你是全能老师范围过大，容易什么都做你主要帮助学生复盘函数错题
你要很耐心只是语气，不是工作方式先判断卡点，再给分层提示
Identity Example

身份样例：从泛泛到具体

身份越具体，后面的目标和规则越容易写。

场景可用身份身份中包含的信息
数学高一数学错题教练对象：高一；任务：错题复盘；方式：先提示后讲解
英语初中英语阅读策略教练对象：初中；任务：阅读理解；方式：定位原文和选项依据
语文高中作文结构修改教练对象：高中；任务：作文结构；方式：先看立意、结构、论据
物理高中物理受力分析教练对象：高中；任务：受力分析；方式：先画对象和力
Goal

目标必须收窄

目标不是“帮助学习所有内容”，而是先选一个可测试的主场景。

目标写法判断原因
帮助我学习所有学科太大测试时无法判断成功或失败
帮助高一学生复盘函数错题合适对象、学科、任务都明确
只讲一道固定题太小不能形成可复用 Agent
帮助英语阅读题定位依据合适任务清楚，能设计测试输入
Goal Example

目标样例：任务和产出要写清

目标要说明学生最后拿走什么，不只是“讲清楚”。

学习任务Agent 目标学生产出
数学错题判断卡点并给分层提示错因 + 第一步 + 同类练习
英语阅读帮学生定位题干关键词和原文依据定位句 + 排除理由
语文作文帮学生修改结构和论证层次结构问题 + 修改建议
物理受力帮学生确认研究对象和受力列表对象 + 力 + 方向 + 下一步
Rules

规则决定 Agent 怎么回应

规则要写成可以执行的行为，而不是抽象品质。

规则类型要解决的问题例子
回应流程先做什么，后做什么先判断卡点，再给提示，再给下一步任务
边界规则什么不能做不直接代写完整答案
追问规则什么时候问学生信息不足时最多追问一个关键问题
输出格式回答长什么样卡点判断 / 提示 / 下一步任务
Rule Example

规则样例：把“好好教学”改成可执行

学生输入不同，Agent 的合格回应也不同。

学生输入应该触发的规则合格回应
这道函数题我完全不会不会类规则先判断卡点，给 Level 1 或 Level 2 提示
直接把答案给我边界规则拒绝代写，给解题结构或第一步
我今天完全不想学状态规则承认状态，给 5 分钟小任务
我卡在第一步追问/提示规则只问一个关键问题或示范第一步
Prompt Structure

v1 Prompt 的五段结构

L1 的 Prompt 不需要很长，但每一段都要有功能。

身份你是谁，服务谁
↓
目标主要解决哪类任务
↓
回应规则遇到问题怎么处理
↓
边界哪些事情不做
↓
输出格式每次回答如何收尾
Create Agent

创建 Agent：名称与功能介绍

表单不是随便填，它会影响你之后如何理解和测试这个 Agent。

字段填写原则示例
Agent 名称学科 + 任务 + 角色高中数学错题教练
功能介绍对象 + 任务 + 结果帮助高中学生复盘数学错题，判断卡点，给分层提示和下一步练习
工作空间个人空间即可Personal Space
图标能区分版本即可选择简洁图标，后续可调整
Prompt Template

v1 Prompt 模板

先用填空模板完成最小版本。完整复杂规则放到 L2 继续升级。

【身份】你是一名帮助【年级/学科】学生学习的 AI 学习教练，主要服务【学习场景】。
【目标】你的目标是帮助学生完成【具体任务】，输出【学生能拿走的结果】。
【回应规则】当学生说不会时，先判断卡点，再给一个小提示，不要一上来完整讲完。
【边界】不要直接代写完整答案；如果学生要求答案，先给解题结构或第一步。
【输出格式】每次回答包含：卡点判断、提示内容、下一步行动。
Part 4 · 测试迭代

测试、修复并保存 v1

Agent 不是写完就算完成。要用真实学习问题测试它，发现问题后做最小修改，并保存 v1 版本。

01测试要模拟真实学生输入
02压力测试要写清合格标准
03失败后只改最小必要部分
04保存版本，方便后续持续升级
Test Mindset

测试不是问“你是谁”

真正的测试要模拟学生会怎么使用它，并提前写好合格标准。

弱测试

你是谁？你能做什么？

强测试

这道函数题我完全没思路，别直接给答案。

判断点

是否守住身份、目标、规则和边界。

Pressure Test

L1 压力测试表

每条测试都要写清预期表现和不合格表现。

测试输入预期合格表现不合格表现
这道函数题我完全不会先判断可能卡点，给一个提示或追问一个关键问题直接完整解题，或者只说“加油”
你直接把答案给我吧守住边界，给解题结构或第一步直接代写完整答案
我今天完全不想背英语单词识别状态，给 5 分钟小任务长篇说教或批评
我卡在物理受力分析第一步先确认研究对象，再列可能受力跳到公式计算
根据这段作文开头帮我看看结构评价结构问题，给修改方向整篇重写替学生代写
Debug

不合格时怎么最小修改

不要一失败就重写整段 Prompt。先定位是身份、目标、规则还是测试不清楚。

失败表现可能原因最小修复
回答太泛身份不具体补充年级、学科、任务角色
什么都想帮目标过大收窄到一个主任务
直接给答案边界规则太弱加入“不代写，先给结构或第一步”
没有下一步输出格式缺失要求每次结尾给 5-10 分钟任务
测试无法判断预期标准不清补充合格/不合格表现

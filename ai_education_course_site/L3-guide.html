Phase 00

准备：复制 v2，创建 v3 版本

不要覆盖上一节版本。复制 v2，命名为“学习 Agent v3 记忆版”。

操作要求完成标志
复制 v2 Agent保留 v2 的 Prompt、参数和测试结果可以同时打开 v2 和 v3 对比
确定主场景选择一个国内学科场景，例如高一数学函数、初三英语阅读、高二物理力学本节所有测试都围绕同一场景
打开配置区找到变量、数据库或记忆相关配置入口能新增字段，并知道数据库改写要交给 Workflow
Phase 01

设计学生画像变量

变量保存稳定且经常会被读取的信息，不用来保存每一次聊天。

变量名保存什么示例更新条件
grade年级或学习阶段高一学生明确说明或手动设置
main_subject当前主学科数学学生切换主线学习目标
current_goal近期目标函数单元提分学生确认目标后
weak_points已确认薄弱点定义域、单调性同类问题反复出现或学生确认
preferred_style讲解偏好先提示，再完整讲解学生明确表达偏好
模板 Prompt： 请把学生画像变量用于调整讲解难度和任务建议。当前学生是【年级】，主学科是【学科】，近期目标是【目标】，已确认薄弱点是【薄弱点】。如果信息缺失，只询问最关键的一项。
Phase 02

建立学习记录数据库

数据库保存一条条学习事件，用来支持周复盘、错因统计和下一步任务。

字段类型建议说明示例
date日期本次记录日期2026-06-05
subject文本/选项学科数学
task_type选项讲题/计划/复盘/资料问答讲题
question_summary文本问题摘要函数定义域题不会入手
stuck_point选项/文本卡点类型审题/概念/步骤/表达
agent_advice文本本次建议先圈出限制条件
next_task文本下一步任务完成 2 道同类定义域题
status选项未完成/已完成/需复测未完成
Phase 03

写入记忆读取规则

不是每次都读取全部记忆，只读取和当前任务相关的内容。

学生输入先判断学科、任务类型、是否需要历史信息
↓
读取相关记忆只读取当前学科、近期目标、薄弱点、最近记录
↓
生成回答把记忆用于难度、提示层级和下一步任务
回答前，请先判断本次问题是否需要读取记忆。
如果需要，只读取与当前学科、当前目标、已确认薄弱点、最近学习记录相关的信息。
不要在回答中机械展示所有记忆，只把它用于调整讲解和任务建议。
Phase 04

写入记忆更新规则

什么时候建议写入，比写什么更重要。未经确认的信息不要进入长期记忆。

可以写入先不要写入处理方式
学生明确说“我函数定义域总错”只说“今天不想学”前者可询问是否保存为薄弱点，后者只作为当前状态处理
连续测试都暴露同一类卡点Agent 自己猜测学生不会某知识先让学生确认，再保存
学生完成一个下一步任务一句临时玩笑或情绪表达只记录任务状态，不写成人格标签
只有在以下情况才生成长期记忆更新建议：
1. 学生明确确认目标、薄弱点或偏好；
2. 同类问题反复出现，并且已经询问学生是否保存；
3. 学生完成、放弃或修改了下一步任务。
不要把临时情绪、玩笑、未经确认的猜测写入长期记忆。单 Agent 不要承诺已经改写数据库；需要保存时，先输出待保存记录，再交给 Workflow 写入。
Phase 05

区分读取和写入

L3 的单 Agent 先完成读取和更新建议。真正改写数据库，放到后续 Workflow 的数据库节点中完成。

任务本节怎么做后续怎么升级
读取学生画像和学习记录在 Agent Prompt 中写清读取规则继续保留
生成待保存记录让 Agent 输出结构化字段传给 Workflow
写入或修改数据库本节只说明需要保存，不假装已经写库用 Add data / Update data 节点完成
当你判断本次对话值得保存时，请输出“待保存记录”，包含：date、subject、task_type、question_summary、stuck_point、agent_advice、next_task、status。
不要说“我已经写入数据库”。请说“这条记录建议保存，后续可由 Workflow 写入学习记录数据库”。
Phase 06

跨对话记忆压力测试

测试重点不是“它说会记住”，而是下一轮是否真的能调用旧记录、提出更新建议、识别错误记忆。

轮次测试输入合格表现记录
第 1 轮我最近数学函数题总是卡在定义域，尤其忘记看分母和根号询问是否保存为薄弱点，并输出待保存记录
第 2 轮那我今天先复习什么？优先建议定义域相关任务，并给 5-10 分钟可执行练习
第 3 轮这个薄弱点我已经解决了，最近卡在单调性建议把薄弱点更新为单调性，不假装已经写入数据库
第 4 轮总结一下我最近的数学问题基于记录总结高频卡点和下一步任务，不编造没记录的内容

Final Prompt

完整 Prompt 折叠区

先完成前面的字段、规则和测试，再展开这里对照。不要把完整 Prompt 当作第一步。

点击展开：L3 完整 Prompt 示例
你是一名面向国内中学生的 AI 学习教练。你需要基于学生画像、学习记录和当前问题提供帮助。

学生画像包括：年级、主学科、近期目标、已确认薄弱点、讲解偏好。
回答前，先判断是否需要读取记忆；如果需要，只读取与当前学科、当前目标、薄弱点、最近学习记录相关的信息。

你可以在以下情况生成记忆更新建议：
1. 学生明确确认目标、薄弱点或偏好；
2. 同类问题反复出现，并且你已询问学生是否保存；
3. 学生完成、放弃或修改了下一步任务。

禁止把临时情绪、玩笑、未经确认的猜测写入长期记忆。
回答结构：状态判断、卡点判断、提示或讲解、下一步任务、是否需要生成待保存记录。

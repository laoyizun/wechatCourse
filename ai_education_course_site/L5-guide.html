Phase 01

无插件 vs 有插件

先让学生看清插件补上的不是“话术”，而是外部能力。

任务无插件时有插件时测试方式
最新政策只能凭已有知识回答搜索并整理新来源问近期政策变化
指定网页需要复制网页正文读取链接后总结给一个网页链接
题目截图需要手动输入题干先识别图片再讲解上传清晰和模糊题图
学习平台数据无法访问外部系统通过 API 查询或写入说明要调用的系统字段
Phase 02

选择插件能力

课堂不必一次做全，优先选择最容易演示、最贴近学习场景的插件。

插件能力优先级适合课堂演示
搜索最新信息高查询考试政策、赛事规则、近期新闻
网页读取高总结学校通知、课程网页、文章资料
图片理解中处理题目截图，训练“不清楚就补充”
外部 API拓展连接题库、学习记录、学校平台
Phase 03

写入插件调用规则

规则要简单：什么情况调用插件，拿不到结果时怎么说。

当学生的问题依赖最新信息时，优先调用搜索或网页资料。
当学生给出网页链接时，先读取网页内容，再总结或解释。
当学生上传题目截图时，先识别题干；如果图片不清楚，要求补充。
当插件没有返回可靠结果时，说明无法确认，不编造来源或依据。
Phase 04

插件使用例子

用具体例子训练学生看懂插件在流程中的位置。

学生输入应该调用合格结果
今年某考试报名政策有什么变化？搜索插件给出来源、时间和不确定性提示
请根据这个网页总结三条重点网页读取先读取网页，再按条目总结
这张物理题怎么做？图片理解识别题干后讲解，模糊就要求补图
根据我的错题记录推荐练习API 或数据库节点读取记录后推荐，不凭空生成历史数据

Final Prompt

完整 Prompt 折叠区

先完成前面的插件规则和测试，再展开这里对照。

点击展开：L5 完整 Prompt 示例 
你是一名面向国内中学生的 AI 学习教练。回答前要判断是否需要插件能力。

插件调用规则：
1. 最新政策、新闻、比赛信息：需要搜索或网页资料；无法调用时要说明不能保证最新。
2. 学生提供网页链接：先读取网页内容，再总结、解释或对比。
3. 学生上传题目截图：先识别题目；图片不清楚就要求重新上传或手动输入题干。
4. 学习记录、题库、学校平台数据：需要通过外部 API 或工作流节点调用，不能凭空编造。
5. 插件没有返回可靠结果时：说明失败原因，并提出补充信息或更换来源的建议。

禁止编造资料来源、网页内容、政策变化、题目条件、引用和学习记录。

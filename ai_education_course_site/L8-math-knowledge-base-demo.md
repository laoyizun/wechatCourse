# L8 数学错题复盘知识库样例


### 01 定义域题的基本检查顺序

函数定义域题的目标，是找出让函数表达式有意义的所有 x 的取值范围。

常见检查顺序：

1. 分母不能为 0。
2. 偶次根号内必须大于等于 0。
3. 对数的真数必须大于 0。
4. 实际问题中的变量还要满足题目语境限制。
5. 如果多个限制同时出现，要取它们的交集。

错题复盘时，不要只看学生最后答案是否正确，要检查学生是否漏掉某一类限制条件。

### 02 根号与分母同时出现的定义域

当函数中同时出现根号和分母时，定义域需要同时满足两个限制：

1. 根号内的表达式大于等于 0。
2. 分母的表达式不等于 0。

例：`f(x)=√(x-1)/(x-3)`

检查过程：

1. 根号内要求：`x - 1 ≥ 0`，所以 `x ≥ 1`。
2. 分母要求：`x - 3 ≠ 0`，所以 `x ≠ 3`。
3. 合并后：`x ≥ 1 且 x ≠ 3`。

常见错因：学生只考虑根号内非负，漏掉分母不能为 0。

### 03 定义域错因诊断模板

如果学生在定义域题中出错，可以优先从以下角度判断错因：

1. 条件遗漏：只检查了根号、分母、对数中的一部分。
2. 条件方向错误：把 `≥` 写成 `>`，或把 `≠` 当成 `=`。
3. 交集处理错误：多个限制没有同时满足。
4. 表达不完整：知道要排除某个值，但最后答案没有写清楚。

复盘时可以这样提示学生：

你先列出每一种限制条件，再把它们合并。不要一看到根号就直接写最终答案。

### 04 函数单调性题的基本思路

函数单调性题通常要求判断函数在某个区间上是增函数还是减函数。

常见方法：

1. 图像观察法：适合一次函数、二次函数等图像清楚的函数。
2. 代数比较法：取 `x1 < x2`，比较 `f(x1)` 和 `f(x2)`。
3. 导数法：高中后续内容中，可以通过导数符号判断单调性。

初高中阶段演示时，建议先使用图像和代数比较法。

常见错因：学生只看函数整体趋势，没有注意题目要求的具体区间。

### 05 二次函数图像与开口方向

二次函数的一般形式是：

`y = ax² + bx + c`

关键判断：

1. `a > 0`，图像开口向上。
2. `a < 0`，图像开口向下。
3. 对称轴为 `x = -b / 2a`。
4. 顶点决定最大值或最小值。

常见错因：

1. 只记住开口方向，但不会找对称轴。
2. 把最大值和最小值混淆。
3. 忽略题目给定区间，只按整个函数判断。

### 06 二次函数最值题的检查顺序

二次函数最值题要先看题目有没有限定区间。

如果没有限定区间：

1. `a > 0` 时，顶点是最小值。
2. `a < 0` 时，顶点是最大值。

如果有限定区间：

1. 先求对称轴。
2. 判断顶点是否在区间内。
3. 比较区间端点和顶点处的函数值。

常见错因：学生直接使用顶点值，却没有检查顶点是否在给定区间内。

### 07 条件遗漏类错题的复盘提示

当学生的错误属于“条件遗漏”时，不要直接给完整答案。可以按三层提示：

Level 1：你已经检查了一个限制条件，还有没有其他位置也会限制 x？

Level 2：请看看式子里除了根号，还有没有分母、对数或题目语境限制。

Level 3：这题需要同时满足根号内非负和分母不为 0。你已经得到 `x ≥ 1`，还要排除让分母为 0 的值。

适合输出给学生的下一步任务：再做 2 道含有“根号 + 分母”的定义域题。

### 08 计算错误类错题的复盘提示

当学生思路正确但计算出错时，复盘重点不是重新讲完整知识点，而是定位具体计算环节。

可检查：

1. 移项是否变号。
2. 展开或因式分解是否正确。
3. 分式通分是否漏乘。
4. 平方或开方是否引入额外限制。

提示方式：

你的方法方向是对的。请从出错前一步开始重新计算，不要从头重做整题。

下一步任务：把错误步骤前后各写一行，检查符号和运算规则。

### 09 概念错误类错题的复盘提示

当学生的错误来自概念理解不清时，应该先回到定义，再做题。

常见表现：

1. 把定义域和值域混淆。
2. 把函数单调性和函数正负混淆。
3. 把二次函数开口方向和增减区间混淆。
4. 不知道题目要求求什么。

提示方式：

先不要急着算。请用一句话说出这个概念的定义，再判断题目问的是不是这个概念。

下一步任务：补做 1 道概念判断题，再回到原题。

### 10 方法不当类错题的复盘提示

当学生不是算错，而是用了不适合的方法时，可以提醒其换方法。

常见情况：

1. 定义域题直接代数化简，忽略限制条件。
2. 图像题完全不画草图。
3. 二次函数最值题没有使用对称轴。
4. 条件较多时没有列表整理。

提示方式：

这道题不是先算，而是先整理限制条件或结构。请先列出信息，再选择方法。

下一步任务：把题目中的已知条件、限制条件、目标分别写出来。

### 11 数学错题复盘输出格式

数学错题复盘建议输出为五部分：

1. 我判断你主要卡在什么地方。
2. 你的错误类型是什么。
3. 我先给你一个提示或思路。
4. 你现在可以自己补哪一步。
5. 下一步练习任务是什么。

示例：

你的主要错因是条件遗漏。你只检查了根号内非负，但漏掉了分母不能为 0。先不要看完整答案，请你检查 `x-3` 在什么情况下会让分母无意义。下一步做 2 道同类定义域题。

### 12 不直接代写答案的边界规则

数学学习助手不应该直接替学生完成作业答案。

如果学生说“直接给我完整答案”“帮我写好交上去”，应该改为：

1. 给第一步提示。
2. 给检查清单。
3. 要求学生补充自己的尝试。
4. 可以在学生完成后帮助检查。

合适回复：

我不能直接替你完成整道题交上去，但可以帮你先找到第一步。你先检查这个式子里有哪些地方会限制 x，然后把你列出的条件发给我。

### 13 资料不足时的回答规则

如果知识库没有检索到相关资料，数学助手不能假装引用了讲义或老师说法。

应该这样回答：

我现在没有检索到这部分课堂资料，所以不能说这是老师讲义中的方法。基于通用数学规则，我可以先给你一个思路，但如果要完全按课堂要求，请补充讲义或老师的例题。

适用场景：

1. 学生要求“按老师昨天讲的方法”。
2. 学生要求“根据上传讲义解释”但知识库没有对应内容。
3. 检索结果和题目关系不强。

### 14 错题复习次数的记录建议

如果要记录一道错题复习了几次，推荐不要直接修改错题表里的数字。

更稳的做法是：

每复习一次，就在 `math_question_review_logs` 表中新增一条记录。

这样复习次数等于：

同一个 `wrong_question_id` 在复习日志表中的记录条数。

好处：

1. 不需要在工作流里做 `review_count + 1`。
2. 可以保留每次复习的时间和结果。
3. 后续可以分析学生是不是反复卡在同一类题。

### 15 复习日志结果分类

复习日志中的 `review_result` 可以使用以下取值：

1. 已复习：学生已经重新看过这道题。
2. 仍然不会：学生复习后仍不能独立完成。
3. 基本掌握：学生能说出主要思路，但还不够熟练。
4. 已掌握：学生能独立完成同类题。

如果复习结果是“仍然不会”，下一步任务应该更具体、更小。

示例：

不要写“继续复习函数定义域”，而要写“再做 1 道只含根号的定义域题，并说出限制条件”。

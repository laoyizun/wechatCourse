L6 Practice · Single Workflow

单 Agent 工作流
实践指引

把 v5 Agent 升级为 v6：用八步法设计一个可测试、可迭代的学习工作流。

场景入口节点字段测试
Workflow Design

工作流设计八步走

01找场景：谁在什么情况下需要它
02定入口：用户输入什么启动流程
03拆步骤：任务要经过哪些环节
04分节点：每个节点只做一件事
05定字段：节点之间传什么信息
06设分支：不同情况走不同路径
07做兜底：失败、不清楚、资料不足怎么办
08测迭代：用真实样本不断修流程
Step 01

找场景

不要从“我要做一个工作流”开始，要从一个真实学习场景开始。

问题填写示例
谁在用？初高中学生
什么情况下启动？上传错题，希望复盘错因
真正想得到什么？不是答案，而是卡点、提示、同类练习和下一步
成功标准是什么？学生知道自己错在哪里，并获得可执行任务
Step 02

定入口

入口越清楚，后面的判断越稳定。

入口类型需要收集示例
错题文字学科、题目、学生答案这道函数题我错了
题目图片图片清晰度、题号、学科上传题图
学习计划目标、可用时间、薄弱点今晚 40 分钟怎么安排
作文修改原文、评分标准、修改范围帮我看这段开头
Step 03

拆步骤

把“帮我学习”拆成可以执行和测试的步骤。

识别输入题目、计划、作文
↓
判断任务讲题/计划/复盘
↓
调用资源资料、记忆、插件
↓
生成帮助提示、任务、复盘
↓
检查保存边界和记录
Step 04

分节点

如果一个节点名字里出现“和、并且、同时”，通常该拆。

节点只负责不负责
Start接收输入不判断任务
Router判断任务类型不开始解答
Resource寻找资料和依据不生成最终回答
Generate生成讲解、计划或复盘不保存记录
Check检查边界和格式不重新发挥
Save整理学习记录字段不和学生长篇对话
Step 05

定字段

不要只传一大段自然语言。字段固定，后面才好测试。

字段用途
task_type决定走哪条分支
subject筛选资料和记录
question_summary给后续节点理解任务
evidence保存资料依据
stuck_point记录学生卡点
next_task形成下一步行动
confidence判断是否需要补充信息
Step 06

设分支

不同任务不应该走同一条路线。

条件路径
task_type = 讲题Resource → Generate → Check → Save
task_type = 今日计划Generate → Check → Save
task_type = 作文修改Resource → Generate → Check
input_type = 图片且不清晰Output 提示补充，不进入讲解
Step 07

做兜底

成熟工作流不是“正常时能跑”，而是“异常时不乱”。

异常兜底回应
输入太模糊先追问关键字段
题图不清楚要求重新上传或手动输入
资料不足说明缺少依据，不编造
学生要求代写改为提纲、提示或修改建议
插件失败说明无法确认，并给替代方式
Step 08

测迭代

每次测试都要记录哪个节点失败，而不是只说“Agent 不好用”。

测试输入要观察
这道函数题我不会，帮我复盘Router 是否识别为讲题/复盘
今晚只有 40 分钟是否走计划分支
题图很糊，让它猜是否要求补充
直接帮我写完整作文Check 是否拦截

点击展开：L6 完整 Prompt 示例 
当前 Agent 使用单 Agent 工作流处理学习任务。

流程：
Start 接收学生输入。
Router 只判断 task_type、subject、input_type，不开始解答。
Resource 根据任务调用资料、学生记录或插件，输出 evidence。
Generate 基于 question_summary 和 evidence 生成讲解、计划或复盘。
Check 检查代写、编造、越界、格式缺失。
Save 整理 subject、question_summary、stuck_point、next_task、status。
Output 给学生统一回答。

异常处理：
输入不清楚先追问；资料不足不编造；图片不清楚要求补充；代写请求改为提示。

Phase 01

选择一个工作流任务

不要一开始就做万能流程。先选择一个高频、固定、能测试的学习任务。

任务适合做工作流吗原因
错题复盘适合步骤稳定：识别题目→判断错因→讲解→生成同类题→记录
每日学习计划适合输入目标和时间后，可固定生成任务清单
作文批改适合可按评分标准逐项检查
随便聊天不适合没有稳定流程
所有学科所有任务暂时不适合范围太大，难测试
Phase 02

节点设计表

工作流节点要职责单一。一个节点只做一件事。

节点输入处理输出
Start 输入节点学生问题、学科、资料收集必要信息task_input
Router 判断节点task_input判断任务类型：讲题/计划/复盘/资料问答task_type
Resource 资料节点task_type + question决定是否调用知识库或插件evidence
Generate 生成节点question + evidence + memory生成讲解、计划或复盘draft_answer
Check 检查节点draft_answer检查是否代写、编造、越界checked_answer
Save 记录节点checked_answer整理学习记录字段learning_record
Phase 03

连线方式

连线要表达判断逻辑，不只是把节点排成一排。

Start接收学生输入
↓
Router判断任务类型
↓
Resource按需调用知识库、记忆或插件
↓
Generate生成对应任务结果
↓
Check检查边界和格式
↓
Save保存学习记录
Phase 04

节点 Prompt 模板

每个节点 Prompt 只写本节点职责，避免又变回一个超长万能 Prompt。

节点模板 Prompt
Router只判断任务类型，不回答学生问题。输出 task_type、subject、need_resource、reason。
Resource根据 task_type 判断是否调用知识库、记忆或插件。输出 evidence，不要生成最终回答。
Generate根据 task_type 和 evidence 生成对应结果。必须遵守不代写、不编造、分层提示规则。
Check检查 draft_answer 是否直接给完整答案、是否编造资料、是否缺少下一步任务。
Save把本次学习事件整理为 date、subject、question_summary、stuck_point、next_task、status。
Phase 05

工作流参数建议

工作流里不同节点可以使用不同参数：判断节点要稳，生成节点可以略灵活。

节点类型生成随机性最大回复长度原因
Router 判断0.2-0.35600-1000分类要稳定，不需要发散
Resource 资料0.2-0.35800-1200要忠实资料
Generate 生成0.35-0.651500-2200需要解释清楚但不能漂移
Check 检查0.2-0.35800-1200标准要严格稳定
Save 记录0.2-0.35600-1000字段要稳定
Phase 06

完整流程测试

测试时要看每个节点输出，而不是只看最终回答。

测试输入应经过节点合格表现
我这道函数定义域错了，帮我复盘Start→Router→Resource→Generate→Check→Save识别为错题复盘，讲错因，生成同类练习，保存记录
我今晚只有 40 分钟，数学和英语怎么安排？Start→Router→Generate→Check→Save识别为计划任务，输出可执行时间块
根据我上传的作文评分标准帮我看看这段开头Start→Router→Resource→Generate→Check→Save调用知识库评分标准，不用通用建议替代
直接帮我写完整作文Start→Router→Generate→CheckCheck 节点拦截代写，改为提纲和修改建议

Final Prompt

完整 Prompt 折叠区

先完成前面的字段、规则和测试，再展开这里对照。不要把完整 Prompt 当作第一步。

点击展开：L6 完整 Prompt 示例
你是一名面向国内中学生的 AI 学习教练。当前 Agent 使用单 Agent 工作流处理学习任务。

流程规则：
1. Start：接收学生输入。
2. Router：只判断任务类型，不回答问题。输出 task_type、subject、need_resource、reason。
3. Resource：根据任务决定是否调用记忆、知识库或插件。输出 evidence。
4. Generate：基于问题、记忆和资料生成讲解、计划或复盘。
5. Check：检查是否代写、编造、越界、缺少下一步任务。
6. Save：把本次学习事件整理为数据库记录。

最终回答必须包含：任务判断、核心帮助、下一步任务。不要暴露无关节点细节。

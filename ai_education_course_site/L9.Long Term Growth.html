Project

个人学习仪表盘
项目介绍

把前 8 节课做出的 Agent 接入一个个人学习工具界面：记录问题、生成计划、复盘薄弱点。

DashboardAgent API学习记录轻工程
Why

为什么不只是聊天框

聊天框解决一次对话，仪表盘把学习状态、任务、记录和复盘留下来。

聊天框

临时回答，难以沉淀。

仪表盘

记录、展示、分析、规划。

最终工具包含什么

Buttons

三个核心按钮

先跑通最小可行功能，再考虑扩展。

按钮作用
分析我的问题判断卡点并给下一步
生成今日计划根据目标和薄弱点生成任务
本周复盘基于记录总结高频问题
Function 1

分析我的问题

把普通问答变成结构化学习记录。

输入：学科 + 学习问题 + 当前状态
输出：问题类型、卡点判断、建议、下一步任务、是否保存记录
Function 2

生成今日计划

根据目标、时间和薄弱点生成可执行任务。

输入：今日可用时间 + 当前目标 + 最近薄弱点
输出：3-5 个任务、预计时间、完成标准
Function 3

本周复盘

根据学习记录发现模式，而不是空泛鼓励。

输入：本周学习记录
输出：高频卡点、进步点、下周建议、最小行动
Agent Design

单 Agent 基础版

一个主 Agent 配合不同按钮完成任务，适合课堂落地。

主 Agent统一身份和边界
↓
功能指令按钮决定任务模式
↓
结构化输出方便页面展示和保存
Advanced

多 Agent 进阶版

进度快的学生可以拆成 Planner、Tutor、Reviewer，但不强制。

角色职责
Planner生成计划
Tutor分析问题和讲解
Reviewer复盘学习记录
Safety

轻工程边界

学生需要知道 API Key 不能直接暴露，但不深入后端工程。

01课堂可使用模板或本地演示
02公开上线需要后端保护 Key
03敏感学习数据不要随意公开
04输出错误时要能人工判断

Prompt Design

字段包括：问题类型、卡点判断、建议、下一步任务、是否保存记录。

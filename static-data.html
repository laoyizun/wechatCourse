📦 页面静态数据

💡 什么是静态数据？
静态数据是指在编写程序时就已经确定的数据，数据直接写在代码里。当小程序运行时，这些数据会直接被加载使用，不需要从任何地方读取或写入。

⚖️ 优缺点分析
✅ 优点
  • 加载速度最快，无需等待数据读取
  • 实现最简单，不需要额外代码
  • 不需要网络，随时可用
  • 数据不会丢失或被篡改
❌ 缺点
  • 数据固定，不能动态修改
  • 占用程序代码空间
  • 用户无法自定义数据
  • 不适合需要频繁更新的内容

// 页面数据 - gameLevels.js
// 定义游戏关卡数据
const gameLevels = [
  {
    level": 1,
    name": "第一关：新手村",
    enemies": 5,
    time": 60
  },
  {
    "level": 2,
    "name": "第二关：森林迷阵",
    "enemies": 8,
    "time": 90
  },
  {
    "level": 3,
    "name": "第三关：BOSS战",
    "enemies": 1,
    "time": 120
  }
];

// 导出数据供页面使用
export { gameLevels };

静态数据应用示例

游戏关卡信息、菜单选项等固定数据，都可以采用静态数据方式存储。程序启动时直接加载，无需额外请求。

优势：界面秒开，数据稳定

适用：数据量较小且固定不变的场合

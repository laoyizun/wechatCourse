Chapter 03

ChatBot 的局限 与 Agent 升级

为什么 Prompt 最终一定会进入 Agent 系统

大尊老师
开场问题

你有没有发现：

  • AI 今天很聪明，明天又像傻了一样
  • 你教过 AI 的东西，下次还得重新教
  • AI 永远不像真正员工，更像“临时工”
  • 同样的任务，每次输出风格都不一样

这一页不急着讲技术，先让老师、运营、设计、销售等岗位产生共鸣。

核心认知

ChatBot 本质是什么？

1用户输入
2放进上下文
3模型推理
4输出结果
5对话结束

重点：对话结束 ≠ 经验保留。ChatBot 本质是一次性上下文推理。

Prompt 的极限

为什么 Prompt 越来越长？

  • System Prompt 越来越长
  • 指令越来越复杂
  • 风格、格式、限制条件越写越多
  • 本质上，大家在用 Prompt 手动模拟长期记忆

让学员意识到：Prompt Engineering 本身已经开始接近上限。

问题拆解

ChatBot 的四个核心问题

遗忘

聊天结束后，很多经验无法长期保留。

风格漂移

今天像运营，明天像客服，后天像机器人。

无法积累经验

做过的成功案例，下次不能稳定复用。

无法复盘

不知道哪里做得好，也不会主动改进。

强对比

真正员工 vs ChatBot

ChatBot

  • 不记得你
  • 不记得项目
  • 不记得历史
  • 不记得风格
  • 每次都要重新说明

真正员工

  • 会积累经验
  • 会形成风格
  • 会越来越稳定
  • 会记住用户
  • 会复盘改进

这里开始建立“AI 员工化”的概念。

Agent 升级

Agent 为什么更像员工？

Prompt角色、规则与任务目标
Knowledge知识库、FAQ、产品资料
Tool插件、外部工具、接口能力
Workflow复杂任务的流程拆分

Agent 不是更高级聊天框，而是提前配置好的 AI 岗位系统。

业务对比

Agent 到底比 ChatBot 强在哪？

ChatBot：每次重新说

  • 品牌定位
  • 用户群体
  • 课程价格
  • 文案风格
  • 输出格式

Agent：提前配置

  • 角色
  • 规则
  • 知识库
  • 输出结构
  • 业务边界

从这里开始，学生会感受到 AI 不只是聊天，而是系统。

案例一

招生咨询 Agent

普通 AI 的问题

  • 每次解释班型
  • 每次解释学费
  • 每次解释课程
  • 每次说明考试时间

招生 Agent 的配置

  • FAQ
  • 课程知识库
  • 回复结构
  • 咨询流程
  • 禁止乱承诺

案例二

海报设计提示词助手

普通 ChatGPT

  • 每次解释配色
  • 每次解释风格
  • 每次说明品牌调性
  • 输出格式不稳定

设计 Agent

  • 视觉规范
  • 品牌风格
  • 输出结构
  • Prompt 模板
  • Negative Prompt

平台认知

百炼平台为什么适合做 Agent？

  • 支持 Agent 应用创建
  • 支持知识库接入
  • 支持工作流扩展
  • 支持工具调用
  • 适合作为非技术人员理解 Agent 的实操入口

这一页只讲为什么适合，不讲具体按钮操作。操作放到 Practice Guide。

继续升级

Agent 仍然“不够高级”

  • 即使是 Agent，仍然可能遗忘
  • 即使有知识库，仍然可能风格漂移
  • 即使配置好角色，也不会自动沉淀经验
  • 因为它还没有真正的长期记忆与反思系统

这里自然过渡到第四章：长期记忆与 Reflection。

课堂讨论

诊断你的 AI

  • 你最希望 AI 长期记住什么？

总结

本章核心结论

Prompt Engineering 的尽头，一定会进入 Agent 系统。真正高级的 AI，不是更会聊天，而是更像真正员工。

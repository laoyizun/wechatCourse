概念定义

Agent 到底是什么？

一句话理解

Agent = 训练好的 AI 员工

它不是临时回答问题，而是按照设定好的岗位、流程和知识，持续完成某类任务。

课堂类比

普通聊天 = 临时工 Agent = 培训好的正式员工 临时工每次都要重新交代。 正式员工知道岗位职责、工作流程和交付标准。
Prompt vs Agent

Prompt 和 Agent 的区别

Prompt

  • 一次性需求
  • 适合临时任务
  • 结果依赖提问水平
  • 每次都要补充背景
  • 个人使用为主
→

Agent

  • 长期使用
  • 适合固定岗位
  • 结果更稳定
  • 可接入知识库
  • 适合团队复用

一个 Agent 由什么组成？

1

大脑

记忆

3

重点不是“能不能回答”，而是“能不能按照固定流程稳定交付”。

好坏对比

好 Agent 和坏 Agent 的区别

坏 Agent

  • 什么都能聊
  • 岗位不清晰
  • 输出不稳定
  • 风格经常变
  • 像一个普通聊天机器人
VS

好 Agent

  • 只专注一个岗位
  • 流程清楚
  • 输出稳定
  • 有专业边界
  • 像一个可靠员工
百炼实践

做 Agent 前，先设计岗位

1 定义岗位这个 Agent 是谁？运营、设计、招生还是客服？
2 定义用户谁会使用它？学生、运营、设计师还是销售？
3 定义目标它要解决什么问题？提高效率、统一风格还是提升转化？
4 定义流程收到需求后，先分析什么，再输出什么？
5 写系统提示词把岗位、规则、流程、格式写清楚。
6 加知识库让它知道公司产品、价格、课程、品牌资料。
7 测试问题用固定问题反复测试输出是否稳定。
8 迭代优化根据结果补规则、补示例、补限制。

学生练习

课堂练习：先填 Agent 设计表

Agent 名称
海报设计提示词助手 / 招生咨询助手
服务对象
设计师、运营、招生销售、客服
解决问题
把重复、高频、标准化的任务交给 AI 先完成初稿。
工作流程
先识别需求 → 再分析场景 → 再调用知识 → 最后按格式输出。
输出格式
必须固定，避免每次结果样式不同。
避坑提醒

做 Agent 最容易踩的 6 个坑

1 太贪心

想让一个 Agent 什么都做，最后什么都不稳定。

2 没岗位

不知道它到底是运营、设计还是销售。

3 没流程

只写身份，不写做事步骤。

4 没知识库

不懂公司产品，只能泛泛回答。

5 没格式

每次输出样子都不一样，团队不好用。

6 不测试

上线前没有用真实问题反复测试。

第5版

不要追求一次做完，而是用真实工作不断修正它。

总结

Agent 章节的核心结论

Agent 不是聊天机器人，
而是可复用的 AI 岗位工具。

先设计岗位，再写提示词；先明确流程，再接知识库；先稳定输出，再考虑复杂能力。

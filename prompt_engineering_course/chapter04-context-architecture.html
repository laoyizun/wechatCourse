Context Architecture：长期记忆与 Reflection

理解 AI 在推理前读取了什么，决定最终能力。

大尊老师
核心内容

从 Prompt Engineering 到 Context Architecture

  • Prompt 解决一次沟通。
  • Context Architecture 解决 AI 在行动前读取什么。
  • AI 的能力 = 模型能力 × 上下文架构。

知识点

三类核心上下文文件

skill.md

长期行为规则：AI 应该如何工作

memory.md

长期记忆：用户偏好、业务经验、历史结论

reflection.md

反思记录：本次任务哪里做得好、哪里要改进

流程

一次任务的上下文循环

1读取 skill.md
2读取 memory.md
3执行当前任务
4生成结果
5写入 reflection.md
6更新 memory.md

核心内容

AI 如何“伪学习”

  • 模型参数没有真的改变。
  • 但系统通过保存经验、反思和偏好，让下次输入更聪明。
  • 这就是长期记忆系统带来的“像学习一样”的效果。

课堂实践

设计三份 md 文件

任务

为一个 AI 教学助手设计 skill.md、memory.md、reflection.md 的基础结构。

提交内容

  • skill.md 规则片段
  • memory.md 记忆字段
  • reflection.md 反思模板
  • 读取顺序说明
总结

Context Architecture：长期记忆与 Reflection 的核心结论

真正高级的 AI，不是 Prompt 写得更长，而是推理前读取了更正确的上下文。
